# 계획 수립

복잡한 작업을 체계적으로 분석하고 실행 가능한 계획과 Todo 목록 생성.

## 핵심 원칙

- **이해 우선**: 코드 작성 전 기존 패턴과 아키텍처 완전 이해
- **질문 우선**: 모호한 점 발견 시 추측하지 않고 질문
- **다중 옵션**: 하나의 방법만 제시하지 않고 trade-off 다른 여러 옵션 비교
- **검증 필수**: 계획 완료 전 품질 검토

## 워크플로우

| 단계 | 목표 | 산출물 |
|------|------|--------|
| 1. 이해 | 요청 파악 | 요약 확인 |
| 2. 탐색 | 관련 코드 분석 | 핵심 파일 목록 |
| 3. 명확화 | 모호점 해결 | 확정된 요구사항 |
| 4. 설계 | 접근방식 비교 | 선택된 옵션 |
| 5. 계획 | 단계별 계획 | Todo 목록 |
| 6. 검토 | 품질 검증 | 검토된 계획 |

### 특수 상황

| 상황 | 대응 |
|------|------|
| 간단한 작업 | 2-4단계 축소/생략, 바로 계획 수립 |
| 탐색적 요청 | 2단계에 집중, 여러 가능성 탐색 |
| 긴급 요청 | 최소 변경 접근방식 우선 권장 |
| 대규모 작업 | 마일스톤 분할, 첫 마일스톤부터 계획 |

---

### 1단계: 요청 이해

사용자 요청 분석 및 핵심 목표 파악.

**불명확한 경우 질문**:
- 어떤 문제를 해결하려 하는가?
- 기능이 무엇을 해야 하는가?
- 제약 조건/특별 요구사항 존재 여부

**명확한 경우**: 바로 다음 단계로 진행.

---

### 2단계: 코드베이스 탐색

관련 코드와 패턴을 상위/하위 수준 모두에서 이해.

**수행 작업**:

1. 2-3개 sd-code-explorer Agent 병렬 실행 (**`sd-plan/references/exploration-guide.md`** 참조)
   - 유사 기능 구현 추적
   - 관련 영역 아키텍처 매핑
   - 현재 구현 분석 및 확장 지점 파악

2. 각 Agent에게 **핵심 파일 5-10개 목록** 반환 요청

3. Agent 완료 후 식별된 핵심 파일 직접 읽기

4. 발견 내용 정리: 관련 파일, 기존 패턴, 재사용 코드, 제약사항

---

### 3단계: 명확화 질문

설계 전 모든 모호점 해결. **이 단계 건너뛰지 않음**.

**식별 대상**:
- 엣지 케이스 처리 방법
- 오류/예외 처리 전략
- 기존 코드와의 통합 지점
- 범위 경계 (포함/제외 항목)
- 설계 선호도 (성능 vs 가독성 등)

**수행**:
1. 모든 질문을 명확한 목록으로 제시
2. 사용자 답변 대기
3. "알아서 해줘" 응답 시 권장 사항 제시 후 확인

---

### 4단계: 접근방식 설계

trade-off가 다른 여러 구현 방식 설계.

**수행 작업**:

1. 2-3개 sd-code-architect Agent 병렬 실행:

   | 접근방식 | 초점 |
   |----------|------|
   | 최소 변경 | 기존 코드 재사용, 빠른 구현 |
   | 클린 아키텍처 | 유지보수성, 깔끔한 추상화 |
   | 실용적 균형 | 속도 + 품질 조합 |

2. 비교 검토: 작업 규모, 긴급성, 복잡도, 확장성

3. 사용자에게 제시:
   ```
   ## 접근방식 비교

   ### 옵션 A: [이름]
   - 요약: ...
   - 장점/단점: ...
   - 예상 변경 범위: ...

   ### 권장 사항
   [근거와 함께 권장 옵션]
   ```

4. 사용자 선택 대기

---

### 5단계: 계획 수립

선택된 접근방식을 실행 가능한 단계별 계획으로 구체화.

**각 단계 정의**:
- 무엇을 해야 하는가
- 어떤 파일을 수정/생성하는가
- 주의사항 및 예상 복잡도

**작업 유형별 템플릿**: `sd-plan/references/planning-templates.md` 참조.

TodoWrite 도구로 Todo 목록 등록.

---

### 6단계: 계획 검토

계획의 품질과 완전성 검증.

**sd-plan-reviewer Agent 실행**:
- 완전성: 모든 필요 단계 포함 여부
- 순서/의존성: 단계 간 순서 정확성
- 누락 케이스: 엣지 케이스, 오류 처리, 롤백 계획
- 실현 가능성: 각 단계 수행 가능 여부

발견된 문제 수정 후 최종 계획 제시.
