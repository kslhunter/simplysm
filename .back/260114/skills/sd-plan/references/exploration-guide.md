# 코드베이스 탐색 가이드

sd-code-explorer Agent를 효과적으로 활용하기 위한 상세 가이드.

## Agent 프롬프트 템플릿

### 유사 기능 탐색

```
[기능명]과 유사한 기능을 찾아 구현을 포괄적으로 추적하세요.

분석 내용:
- 유사 기능의 진입점과 흐름
- 사용된 패턴과 추상화
- 데이터 흐름과 상태 관리
- 오류 처리 방식
- 테스트 구조

읽어야 할 핵심 파일 5-10개 목록 포함.
```

### 아키텍처 매핑

```
[영역명]의 아키텍처와 추상화를 매핑하세요.

분석 내용:
- 모듈 구조와 의존성
- 핵심 인터페이스와 타입
- 레이어 구분 (UI/비즈니스/데이터)
- 확장 지점과 플러그인 구조
- 설정 및 환경 관리

읽어야 할 핵심 파일 5-10개 목록 포함.
```

### 현재 구현 분석

```
[기능/영역]의 현재 구현을 분석하세요.

분석 내용:
- 현재 동작 방식
- 강점과 약점
- 기술 부채 또는 개선 필요 영역
- 변경 시 영향 범위
- 의존하는 외부 시스템

읽어야 할 핵심 파일 5-10개 목록 포함.
```

### UI/UX 패턴 분석

```
[영역]의 UI 패턴과 사용자 경험을 분석하세요.

분석 내용:
- 컴포넌트 구조와 재사용성
- 상태 관리 패턴
- 이벤트 처리 방식
- 스타일링 접근법
- 접근성 고려사항

읽어야 할 핵심 파일 5-10개 목록 포함.
```

### 테스트 구조 분석

```
[영역]의 테스트 접근 방식을 분석하세요.

분석 내용:
- 테스트 유형 (단위/통합/E2E)
- 테스트 커버리지
- 모킹 전략
- 테스트 데이터 관리
- CI/CD 통합

읽어야 할 핵심 파일 5-10개 목록 포함.
```

## Agent 조합 전략

### 새 기능 추가

| Agent | 역할 |
|-------|------|
| Agent 1 | 유사 기능 탐색 |
| Agent 2 | 관련 영역 아키텍처 매핑 |
| Agent 3 | 통합 지점 분석 |

### 기존 기능 수정

| Agent | 역할 |
|-------|------|
| Agent 1 | 현재 구현 분석 |
| Agent 2 | 영향 받는 영역 매핑 |
| Agent 3 | 관련 테스트 구조 분석 |

### 리팩토링

| Agent | 역할 |
|-------|------|
| Agent 1 | 현재 아키텍처 분석 |
| Agent 2 | 대상 패턴 예시 탐색 |
| Agent 3 | 의존성 및 영향 범위 분석 |

### 버그 수정

| Agent | 역할 |
|-------|------|
| Agent 1 | 버그 발생 영역 코드 추적 |
| Agent 2 | 관련 데이터 흐름 분석 |
| Agent 3 | 유사 버그 수정 이력 탐색 |

## 결과 활용

### Agent 결과에서 확인 사항

1. **핵심 파일 목록**: 직접 읽어 상세 컨텍스트 구축
2. **발견된 패턴**: 새 코드에 적용할 일관된 접근법
3. **주의사항**: 피해야 할 안티패턴이나 제약사항
4. **재사용 가능 코드**: 직접 활용 가능한 기존 코드
5. **통합 지점**: 새 코드가 연결될 위치

### 정보 종합 템플릿

```markdown
## 코드베이스 탐색 결과

### 발견된 패턴
- 패턴 1: [설명 및 예시 파일]
- 패턴 2: ...

### 재사용 가능한 코드
- [파일:줄번호] - [용도]

### 주의사항
- [제약사항 또는 피해야 할 것]

### 핵심 파일 (직접 읽기 필요)
1. path/to/file1.ts - [이유]
2. path/to/file2.ts - [이유]
...

### 통합 지점
- [새 코드가 연결될 위치와 방법]
```

## 탐색 깊이 조절

| 복잡도 | Agent 수 | 핵심 파일 | 분석 범위 |
|--------|----------|-----------|-----------|
| 낮음 (간단한 작업) | 1개 | 3-5개 | 직접 관련 코드만 |
| 중간 (일반 작업) | 2-3개 | 5-10개 | 관련 영역까지 |
| 높음 (복잡한 작업) | 3-4개 | 10-15개 | 간접 영향 영역, 테스트 포함 |
