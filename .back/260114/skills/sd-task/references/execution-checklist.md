# 실행 체크리스트

작업 실행 단계별 확인 항목.

## 시작 전

### 컨텍스트

- Todo 목록 상태 확인
- 이전 계획/접근방식 확인
- 관련 파일 목록 확인

### 준비

- 수정 대상 파일 읽기 완료
- 참조 파일 읽기 완료
- 프로젝트 컨벤션 확인 (CLAUDE.md 등)
- 작업 범위 명확화

## 코드 작성

### 기본 품질

- 구문 오류 없음
- 타입 오류 없음
- import 누락 없음
- 미사용 변수/import 없음

### 스타일 일관성

- 네이밍 규칙 준수
- 들여쓰기 일관성
- 코드 포맷팅 규칙 준수
- import 순서/그룹핑 준수

### 코드 품질

- 단순하고 읽기 쉬운 코드
- 중복 코드 없음 (DRY)
- 적절한 함수/모듈 분리
- 과도한 추상화 없음

### 오류 처리

- 예외 상황 처리
- null/undefined 체크
- 적절한 에러 메시지
- 실패 시 복구 로직 (필요시)

### 보안

- 입력 검증
- SQL 인젝션 방지
- XSS 방지
- 민감 정보 노출 없음

## 언어별 항목

### TypeScript

- 타입 어노테이션 적절 사용
- `any` 타입 최소화
- 인터페이스/타입 정의
- strict 모드 호환

### Angular

- 컴포넌트 구조 일관성
- 서비스 주입 패턴 준수
- Input/Output 데코레이터 적절 사용
- trackBy 함수 적절 사용

## 테스트

### 기존 테스트

- 관련 테스트 실행
- 모든 테스트 통과
- 회귀 테스트 확인

### 새 테스트 (필요시)

- 핵심 기능 테스트
- 엣지 케이스 테스트
- 에러 케이스 테스트

### 프로젝트 검증

- 프로젝트 CLAUDE.md의 빌드/린트/테스트 명령어 참조
- 프로젝트별 컨벤션 및 규칙 확인

## 완료 시

### 코드 확인

- 모든 변경사항 저장
- 디버그 코드 제거
- console.log/print 제거
- 주석 처리된 코드 제거

### 진행 관리

- Todo 항목 상태 업데이트
- 작업 결과 요약 작성
- 다음 단계 안내

## 문제 발생 시

### 오류 발생

1. 오류 메시지 정확히 파악
2. 관련 코드 확인
3. 원인 분석
4. 수정 적용
5. 재검증

### 계획과 다른 상황

1. 차이점 명확히 파악
2. 영향 범위 분석
3. 선택지 정리
4. 사용자에게 설명
5. 결정 대기

### 예상보다 복잡한 경우

1. 복잡도 원인 파악
2. 단순화 가능 여부 검토
3. 작업 분할 고려
4. 필요시 sd-plan으로 재계획
