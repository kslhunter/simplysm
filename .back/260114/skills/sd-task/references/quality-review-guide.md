# 품질 검토 기준

품질 검토 기준.

## 검토 관점

### 1. 단순성/가독성

- 코드 가독성 확인
- 변수/함수명 명확성 확인
- 불필요한 복잡성 제거
- 로직 흐름 이해 용이성

### 2. DRY

- 중복 코드 제거
- 공통 로직 적절한 추출
- 과도한 추상화 방지

### 3. 버그/정확성

- 로직 오류 점검
- null/undefined 처리 확인
- 엣지 케이스 처리 확인
- 비동기 처리 정확성

**일반적 버그 패턴**: Off-by-one, null 참조, 레이스 컨디션, 타입 불일치, 경계 조건 미처리

### 4. 오류 처리

- 예외 적절한 처리
- 에러 메시지 유용성
- 실패 시 복구 가능성
- 에러 적절한 전파

### 5. 보안

- 입력 검증
- SQL/명령어 인젝션 위험
- XSS 취약점
- 민감 정보 노출
- 인증/인가 체크

### 6. 프로젝트 컨벤션

- 네이밍 규칙 준수
- 코드 스타일 일관성
- import 패턴 준수
- 폴더/파일 구조 준수

## 심각도 분류

| 등급 | 기준 | 대응 |
|------|------|------|
| Critical | 보안 취약점, 데이터 손실, 주요 기능 오작동 | 즉시 수정 |
| Major | 버그 가능성 높음, 성능 문제, 유지보수 어려움 | 수정 권장 |
| Minor | 스타일 불일치, 작은 최적화, 문서화 개선 | 선택적 개선 |

## Agent 호출

Task 도구로 sd-code-reviewer Agent 호출:

```
subagent_type: sd-code-reviewer
prompt: "<파일 경로>를 다음 관점에서 검토: ..."
```

### 일반 검토 프롬프트

```
다음 관점에서 코드 검토:
1. 단순성/DRY/가독성
2. 버그/기능적 정확성
3. 프로젝트 컨벤션 준수

신뢰도 80 이상인 이슈만 보고.
```

### 보안 집중

```
보안 관점에서 코드 검토:
- 입력 검증
- 인젝션 취약점
- 인증/인가
- 민감 정보 처리

OWASP Top 10 기준 평가.
```

### 성능 집중

```
성능 관점에서 코드 검토:
- 불필요한 연산
- N+1 쿼리
- 메모리 누수
- 비효율적 알고리즘

실제 성능 영향 있는 이슈만 보고.
```

## 검토 결과 활용

| 등급 | 처리 |
|------|------|
| Critical | 즉시 수정 → 재검증 → 완료 확인 |
| Major | 수정 → 테스트 → 사용자 알림 |
| Minor | 제안으로 나열 → 사용자 결정 대기 |

## 자체 vs Agent 검토

| 자체 검토 적합 | Agent 검토 적합 |
|----------------|-----------------|
| 작은 변경사항 | 복잡한 로직 |
| 명확한 수정 | 여러 파일 변경 |
| 빠른 확인 필요 | 보안 민감 코드 |
| - | 품질 확신 필요 |
