# 작업 실행

계획된 작업을 체계적으로 실행. Todo 목록 기반 순차 처리 및 품질 관리.

## 핵심 원칙

- **계획 기반**: 승인된 계획/Todo를 따름. 범위 임의 변경 금지
- **단계별 진행**: 한 번에 하나의 Todo 집중. 완료 후 다음으로 이동
- **품질 우선**: 중요 변경 후 필요시 품질 검토 수행
- **컨벤션 준수**: 기존 패턴과 스타일 엄격 준수
- **진행 추적**: TodoWrite로 실시간 상태 업데이트

## 워크플로우

### 1단계: 작업 확인

현재 상태와 다음 할 일 파악.

**수행 작업**:

1. Todo 목록 상태 확인
   - 완료/진행 중/대기 중 항목 파악
   - `in_progress` 있으면 해당 항목 계속
   - 없으면 첫 번째 `pending` 선택

2. 작업 컨텍스트 파악
   - 이전 대화의 계획 존재 여부
   - 선택된 접근방식
   - 관련 파일 목록

**계획 없는 경우**: 간단하면 바로 진행, 복잡하면 sd-plan 사용 권장.

**작업 크기 판단 기준**:

| 구분 | 기준 | 대응 |
|------|------|------|
| 간단 | 단일 파일, 명확한 변경, 의존성 없음 | 바로 진행 |
| 보통 | 2-3개 파일, 로직 변경, 일부 의존성 | Todo 작성 후 진행 |
| 복잡 | 4개 이상 파일, 아키텍처 영향, 다수 의존성 | sd-plan 권장 |


### 2단계: 준비

작업 실행에 필요한 컨텍스트 구축.

**수행 작업**:

1. 현재 Todo를 `in_progress`로 변경
2. 필요 파일 읽기: 수정 대상, 참조 파일, 테스트 파일
3. 작업 범위 명확화: 무엇을, 어디서, 주의사항

### 3단계: 실행

실제 코드 작성/수정 수행.

**수행 작업**:

1. 계획된 변경사항 구현
2. 코드베이스 컨벤션 준수
3. 과도한 엔지니어링 방지 (요청된 것만 구현)

상세 체크리스트는 **`sd-task/references/execution-checklist.md`** 참조.

## 4단계: 검증

작성한 코드의 정확성 확인.

**수행 작업**:

1. 기본 검증: 구문/타입/import 오류 없음
2. 기능 검증: 의도대로 동작, 엣지 케이스 처리, 기존 기능 유지
3. 테스트 (해당 시): 기존 테스트 실행, 필요시 새 테스트 작성

**검증 명령어**: 프로젝트 CLAUDE.md의 빌드/린트/테스트 명령어 참조.

**실패 시**: 즉시 수정 후 재검증.

---

## 5단계: 품질 검토 (선택)

코드 품질 전문 검토.

**수행 시점**:

| 구분 | 상황 |
|------|------|
| 필수 | 중요 비즈니스 로직 변경, 보안 관련 코드, 여러 파일 변경, 사용자 요청 |
| 권장 | 각 단계 완료 후, 복잡한 알고리즘 구현, 새 패턴 도입 |
| 선택 | 단순 수정 (오타, 포맷팅), 설정 파일 변경, 문서 업데이트 |

**수행 작업**:

1. Task 도구로 sd-code-reviewer Agent 호출:
   ```
   subagent_type: sd-code-reviewer
   prompt: "<파일 경로>를 다음 관점에서 검토:
           단순성/DRY/가독성, 버그/기능적 정확성,
           프로젝트 컨벤션 준수"
   ```

2. 이슈 분류 및 처리:
   - Critical: 즉시 수정
   - Major: 수정 권장
   - Minor: 선택적 개선

3. **수정 발생 시 4단계(검증) 재수행**

4. 검토 결과 사용자에게 요약

상세 검토 기준은 **`sd-task/references/quality-review.md`** 참조.

## 6단계: 완료 및 전환

현재 작업 마무리 및 다음 이동.

**수행 작업**:

1. 현재 Todo를 `completed`로 변경
2. 작업 결과 요약: 완료 내용, 수정 파일, 주요 변경사항
3. 다음 단계 안내: 남은 Todo 수, 다음 작업 내용

**모든 Todo 완료 시**: 최종 요약 제공.

```markdown
## 작업 완료 요약

### 구현된 내용
- [완료된 기능/변경사항]

### 수정된 파일
- `file1.ts` - [변경 내용]

### 주요 결정사항
- [구현 중 내린 결정]

### 권장 다음 단계
- [추가 작업 필요 시]
```

## 특수 상황

| 상황 | 대응 |
|------|------|
| 계획 없이 시작 | 간단하면 진행, 복잡하면 sd-plan 권장 |
| 계획 변경 필요 | 작업 중단, 이유 설명, 선택지 제공 |
| 예상치 못한 문제 | 문제 설명, 해결 방안 제시, 사용자 결정 대기 |
| 대규모 작업 | 현재 마일스톤 집중, 완료 후 다음 이동 |

**대규모 작업 진행률 보고**:

- 매 Todo 완료 시 진행 상황 요약 (예: "3/10 완료")
- 주요 마일스톤 도달 시 중간 보고
- 예상보다 오래 걸리면 사용자에게 상황 공유
