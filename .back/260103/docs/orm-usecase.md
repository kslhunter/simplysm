# ORM 통합 라이브러리 유즈케이스

> 본 문서는 ORM 라이브러리가 제공해야 하는 기능을 정의합니다.

---

## 1. 핵심 원칙

### 1.1 DB 독립성
- 설정 하나만 변경하여 DB 종류를 전환할 수 있어야 함
- 지원 DB: MySQL 8.0.14+, PostgreSQL 9.0+, MSSQL 2012+

### 1.2 타입 안전성
- 컴파일 타임에 쿼리 구조를 검증할 수 있어야 함
- 런타임에 결과를 올바른 타입으로 변환할 수 있어야 함

### 1.3 실행 환경
- Node.js에서 직접 DB에 연결할 수 있어야 함
- 브라우저에서 서버를 경유하여 DB에 접근할 수 있어야 함
- 두 환경에서 사용자의 ORM 사용법은 동일해야 함

---

## 2. 스키마 정의

### 2.1 테이블 정의
- 테이블 이름을 지정할 수 있어야 함
- 데이터베이스를 지정할 수 있어야 함 (선택)
  - 미지정시 커넥션에 설정된 기본 DB 사용
- 스키마를 지정할 수 있어야 함 (선택)
  - 미지정시 커넥션에 설정된 기본 스키마 사용
- 테이블 설명을 지정할 수 있어야 함 (선택)

#### 2.1.1 컬럼 정의
- 컬럼 이름을 지정할 수 있어야 함
- 데이터 타입을 지정할 수 있어야 함
- NULL 허용 여부를 지정할 수 있어야 함
- 기본값을 지정할 수 있어야 함
- 자동 증가(AUTO INCREMENT)를 지정할 수 있어야 함
- Primary Key를 지정할 수 있어야 함
- 복합 Primary Key의 순서를 지정할 수 있어야 함
- Primary Key 정렬 방향(ASC/DESC)을 지정할 수 있어야 함
- 컬럼 설명을 지정할 수 있어야 함 (선택)

### 2.2 뷰 정의
- 뷰 이름을 지정할 수 있어야 함
- SELECT 쿼리로 뷰를 정의할 수 있어야 함

### 2.3 저장 프로시저 정의
- 프로시저 이름을 지정할 수 있어야 함
- 파라미터를 정의할 수 있어야 함 (컬럼정의외 동일)
- 프로시저 본문을 지정할 수 있어야 함
- 결과를 정의할 수 있어야 함

### 2.4 관계 정의

#### 2.4.1 Foreign Key
- 다른 테이블을 참조하는 FK를 정의할 수 있어야 함
- 복수 컬럼으로 FK를 정의할 수 있어야 함

#### 2.4.2 Foreign Key Target (역참조)
- FK의 반대 방향 참조를 정의할 수 있어야 함
- 1:1 또는 1:N 관계를 구분할 수 있어야 함

#### 2.4.3 Reference Key (논리적 참조)
- DB에 FK 제약조건 없이 논리적 참조를 정의할 수 있어야 함

#### 2.4.4 Reference Key Target (논리적 역참조)
- DB에 FK 제약조건 없이 논리적 역참조를 정의할 수 있어야 함

### 2.5 인덱스 정의
- 인덱스 이름을 지정할 수 있어야 함
- 인덱스 컬럼을 지정할 수 있어야 함
- 복수 컬럼 인덱스를 정의할 수 있어야 함
- UNIQUE 인덱스를 정의할 수 있어야 함
- 정렬 방향(ASC/DESC)을 지정할 수 있어야 함

### 2.6 데이터 타입
- 문자열 (가변 길이, 고정 길이, 대용량 텍스트)
- 숫자 (정수, 소수점)
- 불리언
- 날짜/시간 (날짜+시간, 날짜만, 시간만)
- UUID
- 바이너리

## 3. 연결 관리

### 3.1 연결
- 데이터베이스에 연결할 수 있어야 함
- 연결을 종료할 수 있어야 함
- 연결 상태를 확인할 수 있어야 함

### 3.2 트랜잭션
- 트랜잭션을 시작할 수 있어야 함
- 트랜잭션을 커밋할 수 있어야 함
- 트랜잭션을 롤백할 수 있어야 함
- 예외 발생 시 자동으로 롤백되어야 함
- 트랜잭션 없이 연결할 수 있어야 함

### 3.3 트랜잭션 격리 수준
- DB표준의 트랜잭션 격리 수준을 공통된 방법으로 지정할 수 있어야 함

### 3.4 커넥션 풀
- 커넥션을 재사용할 수 있어야 함
- 유휴 커넥션을 자동으로 정리해야 함
- 손상된 커넥션을 자동으로 재생성해야 함

### 3.5 타임아웃
- 연결 타임아웃을 설정할 수 있어야 함
- 쿼리 실행 타임아웃을 설정할 수 있어야 함

---

## 4. 조회 (SELECT)

### 4.1 기본 조회
- 컬럼 미설정시, 테이블의 모든 컬럼을 조회할 수 있어야 함
- 특정 컬럼만 선택하여 조회할 수 있어야 함
- 컬럼에 별칭을 지정할 수 있어야 함
- 표현식 결과를 조회할 수 있어야 함

### 4.2 조건 (WHERE)
- 조건을 지정하여 필터링할 수 있어야 함
- 여러 조건을 AND로 결합할 수 있어야 함
- 여러 조건을 OR로 결합할 수 있어야 함

### 4.3 정렬 (ORDER BY)
- 오름차순 정렬할 수 있어야 함
- 내림차순 정렬할 수 있어야 함
- 복수 컬럼으로 정렬할 수 있어야 함

### 4.4 제한 (LIMIT/TOP/TABLESAMPLE)
- 상위 N개만 조회할 수 있어야 함
- 오프셋과 개수를 지정할 수 있어야 함 (페이지네이션)
- 무작위 샘플을 조회할 수 있어야 함

### 4.5 중복 제거 (DISTINCT)
- 중복 행을 제거할 수 있어야 함

### 4.6 그룹화 (GROUP BY)
- 컬럼으로 그룹화할 수 있어야 함
- 복수 컬럼으로 그룹화할 수 있어야 함

### 4.7 그룹 조건 (HAVING)
- 그룹에 조건을 지정할 수 있어야 함

### 4.8 조인 (JOIN)

#### 4.8.1 기본 조인
- 다른 테이블과 조인할 수 있어야 함
- 조인 조건을 지정할 수 있어야 함
- 결과를 배열로 받을 수 있어야 함 (1:N)
- 결과를 단일 객체로 받을 수 있어야 함 (N:1, 1:1)

#### 4.8.2 중첩 조인 (Nested Join)
- 조인된 테이블에서 다시 조인할 수 있어야 함
- 무제한 깊이의 중첩 조인을 지원해야 함

#### 4.8.3 다중 조인 (Multiple Join)
- 하나의 테이블에서 여러 테이블을 동시에 조인할 수 있어야 함

#### 4.8.4 FK 기반 자동 조인
- 정의된 FK 관계를 기반으로 자동 조인할 수 있어야 함

### 4.9 서브쿼리
- SELECT 절에 서브쿼리를 사용할 수 있어야 함
- FROM 절에 서브쿼리를 사용할 수 있어야 함
- WHERE 절에 서브쿼리를 사용할 수 있어야 함
- 그 외에도 표준 SQL에 따라 각 부분에 서브쿼리가 가능해야함.

### 4.10 UNION
- 여러 쿼리 결과를 합칠 수 있어야 함
- UNION ALL을 지원해야 함

### 4.11 PIVOT / UNPIVOT
- 행을 열로 변환할 수 있어야 함 (PIVOT)
- 열을 행으로 변환할 수 있어야 함 (UNPIVOT)

### 4.12 행 잠금
- 조회 시 행을 잠글 수 있어야 함 (FOR UPDATE)

### 4.13 검색
- 여러 컬럼에서 텍스트를 검색할 수 있어야 함
- 검색어를 AND로 결합할 수 있어야 함
- 제외 검색을 할 수 있어야 함

## 5. 삽입 (INSERT)

### 5.1 기본 삽입
- 레코드를 삽입할 수 있어야 함
- 삽입된 값을 반환받을 수 있어야 함 (OUTPUT)

### 5.2 조건부 삽입
- WHERE문의 결과가 존재하지 않을 때만 삽입할 수 있어야 함

### 5.3 SELECT 결과 삽입
- SELECT 결과를 다른 테이블에 삽입할 수 있어야 함

### 5.4 대량 삽입
- 다수의 레코드를 효율적으로 삽입할 수 있어야 함

### 5.5 FK 체크 비활성화 삽입
- FK 제약조건을 일시적으로 비활성화하고 삽입할 수 있어야 함

### 5.6 Identity Insert
- 자동 증가 컬럼에 직접 값을 지정하여, 지정값 삽입이 가능해야함

---

## 6. 수정 (UPDATE)

### 6.1 기본 수정
- 조건에 맞는 레코드를 수정할 수 있어야 함
- JOIN을 포함한 조건 설정이 가능해야함.
- 수정된 값을 반환받을 수 있어야 함 (OUTPUT)

### 6.2 표현식으로 수정
- 현재 값을 기반으로 수정할 수 있어야 함 (예: count = count + 1)

---

## 7. 삭제 (DELETE)

### 7.1 기본 삭제
- 조건에 맞는 레코드를 삭제할 수 있어야 함
- JOIN을 포함한 조건 설정이 가능해야함.
- 삭제된 값을 반환받을 수 있어야 함 (OUTPUT)

### 7.2 전체 삭제 (TRUNCATE)
- 테이블의 모든 데이터를 빠르게 삭제할 수 있어야 함

---

## 8. UPSERT (INSERT or UPDATE)

### 8.1 기본 UPSERT
- 존재하면 수정, 없으면 삽입할 수 있어야 함
- 조건을 기반으로 판단해야 함 (PK판단 아님)
- 결과를 반환받을 수 있어야 함 (OUTPUT)

### 8.2 대량 UPSERT
- 다수의 레코드를 효율적으로 UPSERT할 수 있어야 함

---

## 9. 비교 연산

### 9.1 값 비교
- 같음 (=)
- 다름 (<>)
- 초과 (>)
- 이상 (>=)
- 미만 (<)
- 이하 (<=)
- 범위 (BETWEEN)
- NULL-safe여야함 (NULL도 값인것 처럼 비교되도록)

### 9.2 NULL 체크
- NULL 여부 확인
- NOT NULL 여부 확인

### 9.3 Boolean 체크
- TRUE 여부 확인
- FALSE 여부 확인

### 9.4 문자열 패턴
- LIKE 패턴 매칭
- 포함 여부 확인
- 시작 문자열 확인
- 끝 문자열 확인
- 정규식 매칭

### 9.5 IN 연산
- 값 목록에 포함 여부 확인
- 서브쿼리 결과에 포함 여부 확인
- NULL-safe여야함 (NULL도 값인것 처럼 비교되도록)

### 9.6 존재 여부
- 서브쿼리 결과 존재 여부 확인 (EXISTS)

## 17. Raw Query

### 17.1 기본 실행
- SQL 문자열을 직접 실행할 수 있어야 함
- 파라미터 바인딩을 지원해야 함
- SELECT, INSERT, UPDATE, DELETE, DDL 등 모든 쿼리 실행 가능해야 함
- 실행 결과를 반환받을 수 있어야 함
- 트랜잭션 컨텍스트에서 실행할 수 있어야 함

---

## 18. 결과 처리

### 17.1 타입 변환
- 날짜/시간 문자열을 DateTime 객체로 변환
- 날짜 문자열을 DateOnly 객체로 변환
- 시간 문자열을 Time 객체로 변환
- UUID 문자열을 Uuid 객체로 변환
- 숫자를 boolean으로 변환

### 17.2 조인 결과 파싱
- 플랫한 결과를 중첩 객체 구조로 변환
- 1:N 관계를 배열로 변환
- N:1 관계를 단일 객체로 변환
- 무제한 깊이의 중첩 구조 지원

---

## 19. 마이그레이션

- 스키마 변경사항을 버전별로 정의할 수 있어야 함
- 아직 적용되지 않은 버전만 자동으로 적용되어야 함
- 이미 적용된 버전은 다시 실행되지 않아야 함

---

- 연결 실패 시 명확한 에러 메시지 제공

- 타임아웃 에러 처리
