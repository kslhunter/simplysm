## ⚠️ 필수 규칙 (CRITICAL)

1. **정확성 우선**: 코드베이스 분석 후 정확한 정보만 제공 (추측이나 틀린 정보 제공 금지)
2. **독자 맞춤**: 대상 독자(사용자/개발자/AI)에 따라 내용과 수준을 조정하여, 독자가 필요로할 내용만 작성
3. **간결성**: 핵심만 간결하게, 과도한 예시 지양 (마크다운 기본 문법 설명 생략)
4. **논리적 구성**: 정보를 체계적으로 정리, 동일 개념에 동일 용어/패턴 사용
5. **문서 범위**: 기술 문서만 작성 (마케팅 자료, 비기술 문서, 아키텍처 설계, 실제 코드 작성 금지)
6. **한국어 작성**: 전문적 톤 유지

## 트리거

- "문서 작성해줘", "README 만들어줘", "가이드 작성"
- API 문서, 아키텍처 문서, 개발 가이드 요청
- 코드 설명 문서화, CLAUDE.md 작성

## 실행 절차

- [ ] **1. 요구사항 파악**: 문서 유형, 대상 독자, 목적 확인
- [ ] **2. 코드베이스 분석**: 관련 파일, 구조, 패턴 탐색
- [ ] **3. 정보 검증**: 실제 코드와 일치하는지 확인
- [ ] **4. 구조 설계**: 논리적 흐름으로 섹션 구성
- [ ] **5. 초안 작성**: 핵심 내용 + 간결한 예시
- [ ] **6. 검토**: 정확성, 명확성, 일관성 확인
- [ ] **7. 최종 제공**: 한국어 전문적 톤으로 완성
