## ⚠️ 필수 규칙 (CRITICAL)

1. **사용자 명시적 승인 필수**: 계획과 타당성 검토 후 반드시 승인 받고 작업 시작 (승인 없이 작업 시작하거나 파일 저장 금지)
2. **Phase는 기능 단위**: 프로세스 단계(설계→구현→테스트) 아님, 독립적 기능(A기능, B기능, C기능)으로 분리
3. **자동 판단**: 컨텍스트로 레벨(TODO/TASK/PLAN) 자동 선택 (사용자에게 묻지 않음)
4. **의존성 명시**: PLAN 작성 시 Phase 간 의존성과 실행 전략(병렬/순차) 포함
5. **구체적 작성**: 템플릿만 제공하고 내용 비우지 않음, 체크리스트는 구체적으로
6. **모노레포 구조**: `packages/{package}/` 경로 반영, 검증 절차(타입체크→ESLint→테스트) 포함
7. **한국어 작성**: 모든 계획은 한국어로 작성

## 트리거

- "계획", "작업", "PLAN", "TASK", "TODO"
- 코드 작업 (수정/추가/삭제, 기능 구현, 버그 수정, 리팩토링, 파일 생성/삭제 등)

## 작업 레벨 판단 (Decision Tree)

| 레벨     | 기준                                 | 사고 패턴                           | 턴 수      |
| -------- | ------------------------------------ | ----------------------------------- | ---------- |
| **TODO** | compact 없이 한번에 완료             | "바로 시작하면 끝까지 갈 수 있겠다" | ~20턴      |
| **TASK** | 단계별 compact 필요, 하나의 요구사항 | "중간중간 정리하면서 가야겠다"      | 각 20-50턴 |
| **PLAN** | 그외                                 | 그외                                |            |

**판단 로직**:

```
if (compact 없이 한번에 가능) → TODO
else if (단계별 compact 필요) → TASK
else → PLAN
```

## 대규모 PLAN 작성 시 추가 고려사항

**Phase 의존성 분석 및 실행 전략**:

- 독립적: 서로 다른 파일/모듈 → 병렬 가능
- 의존적: B가 A 결과 사용 → 순차 필수
- 예: `Phase 2, 3 → Phase 1 완료 후 병렬 가능`

## 실행 절차

- [ ] **1. 대화 분석**: 작업 크기/복잡도/의존성 파악
- [ ] **2. 레벨 판단**: TODO/TASK/PLAN 자동 선택
- [ ] **3. 계획 작성**: 체크리스트 생성 (PLAN은 의존성 및 실행 전략 포함)
- [ ] **4. 타당성 검토**: 실현 가능성, 누락 사항, 리스크 분석
- [ ] **5. 사용자 설명**: 계획 + 타당성 검토 내용 설명
- [ ] **6. 승인 대기**: 명시적 승인 받을 때까지 대기 ⚠️
- [ ] **7. 사용자 응답 처리**:
  - "저장해줘" → `packages/{package}/PLAN.md` 또는 `TASK.md` 저장
  - "Phase N 세부화" → 해당 Phase 상세화
  - "바로 시작" → 작업 시작
