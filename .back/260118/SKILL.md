# 검토 스킬

프로젝트/패키지의 문제점 및 개선점을 종합 검토한다.

## 작업 흐름

### 1. 범위 결정

전체 프로젝트 또는 특정 패키지

### 2. 관점별 병렬 검토

Task(general-purpose)를 사용하여 다음 관점별로 **각각** 병렬 검토:

- 코드 품질:
  - 코드 품질 전반
  - 표준 기능 및 외부 npm 라이브러리 사용 고려
  - 되도록 `simplysm`의 각 패키지의 기능을 사용하도록 적극 권장 (필요시 의존성 추가)
  - 코딩에 대한 지침 준수 여부 고려
  - 유사한 구현 존재 여부 고려
  - deprecated API 사용 여부 고려
  - 성능 최적화 고려
- 아키텍처:
  - 아키텍처 전반
- 테스트:
  - 테스트 적합성 고려
  - 추가 테스트 고려
  - 단위테스트와 통합테스트 모두 확인
- 문서화:
  - 문서화 전반

**에이전트 프롬프트 템플릿:**

~~~markdown
{{대상}}의 {{관점}}을 검토합니다.

**세부 지침:**
- {{해당 에이전트의 세부 지침}}

**작업 흐름:**
1. 코딩 지침 확인: `.claude/rules/TYPESCRIPT_RULES.md`
2. 세부지침을 통한 검토
3. 결과항목들에 대한 적절성 재검토 (지침: `references/sd-review-validate-guide.md`)

**결과 형식:**
```markdown
## {{대상}} 검토 결과

### 제외 사항

| 제목 | 설명 | 제외이유 |
|------|------|----------|

### 문제점/개선점

**1. 문제 제목 (`파일명:라인번호`)**
- **문제**: {{현재 상태 설명}}
- **제안**: {{구체적 수정 방안}}
```
~~~

### 3. 결과 취합 + 중복 제거

- 4개 에이전트 결과를 하나로 병합
- 동일 위치에 대한 중복 지적 제거
- 유사한 제안 통합

### 4. 건당 병렬 적절성 재검토

Task(general-purpose)를 사용하여 **각각의 문제점/개선점 항목들**을 **각각** 병렬로 재검토:

~~~markdown
다음 지적이 적절한지 지침에 따라 재검토합니다.

**지적:**
{{결과항목 내용}}

**지침:**
- 코딩 지침: `.claude/rules/TYPESCRIPT_RULES.md`
- 적절성 검토 지침: `references/sd-review-validate-guide.md`
~~~

### 5. 결과 출력

재검토 결과를 취합하여 결과 출력:

```markdown
## {{대상}} 검토 결과

### 제외 사항

| 제목 | 설명 | 제외이유 |
|------|------|----------|

### 문제점/개선점

**1. 문제 제목 (`파일명:라인번호`)**
- **문제**: {{현재 상태 설명}}
- **제안**: {{구체적 수정 방안}}
```

- `{{대상}}`: 프로젝트 혹은 패키지명
- `{{구체적 수정 방안}}`: A또는 B라는 식의 여러가지 제안을 하는경우, 무엇이 권장되는지 반드시 표기

**주의:** 사용자의 피드백 없이 임의로 수정진행 절대 금지
