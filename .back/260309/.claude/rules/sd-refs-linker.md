# Reference Guide

Before starting work, **Read** the relevant reference files from `.claude/refs/`.

## Documents

| Situation                                                                              | File to Read                               |
|----------------------------------------------------------------------------------------|--------------------------------------------|
| Writing or editing instruction documents for LLMs                                      | `.claude/refs/sd-prompt-authoring-rules.md` |
| Writing, modifying, or reviewing code                                                  | `.claude/refs/sd-code-conventions.md`       |
| Using or debugging `@simplysm/*` package APIs (including external projects, not just the simplysm monorepo) | `.claude/refs/sd-simplysm-library.md`       |
| Migrating/porting code from another codebase                                           | `.claude/refs/sd-migration.md`              |
| Working with Angular / @simplysm/sd-angular                                            | `.claude/refs/sd-angular.md`                |
| Working with SolidJS / @simplysm/solid / Tailwind                                      | `.claude/refs/sd-solid.md`                  |

## Rules

- Read references **before** starting the related work (not after)
- If work spans multiple areas, you may read multiple references
- If unsure whether a reference applies, read it — the cost of reading is low
