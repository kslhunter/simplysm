# 검토 결과 형식

검토/분석 관련 에이전트의 공통 산출물 형식.

## 심각도 기준

### 코드 검토

| 심각도 | 기준 |
|--------|------|
| Critical | 보안 취약점, 데이터 손실 가능성, 런타임 오류 |
| Warning | 유지보수 저해, 성능 저하, 잠재적 버그 |
| Info | 가독성 개선, 컨벤션 권장, 스타일 제안 |

### 지침 검토

| 심각도 | 기준 |
|--------|------|
| Critical | 필수 항목 누락, 가이드 심각한 위반, 구조적 오류 |
| Warning | 권장 항목 누락, 표현 불명확, 일관성 문제 |
| Info | 스타일 개선, 표현 다듬기, 가독성 향상 |

## 산출물 템플릿

```markdown
## 검토 결과

### Critical issues

**<1. 문제 제목 (`파일명:라인번호`)>**
- **관점**: <관점>
- **문제**: <현재 상태 설명>
- **제안**: <구체적 수정 방안>

### Warning issues

**<2. 문제 제목 (`파일명:라인번호`)>**
- **관점**: <관점>
- **문제**: <현재 상태 설명>
- **제안**: <구체적 수정 방안>

### Info issues

**<3. 문제 제목 (`파일명:라인번호`)>**
- **관점**: <관점>
- **문제**: <현재 상태 설명>
- **제안**: <구체적 수정 방안>
```

## 작성 규칙

- 이슈가 없는 심각도 섹션은 생략
- 이슈가 전혀 없으면 "검토 결과: 이슈 없음" 출력
- 문제 제목은 간결하게 (10단어 이내)
- 파일명:라인번호 형식 필수
- 제안은 구체적이고 실행 가능하게
