### 0. 요청 이해

**목표**: 개선 대상 파악

사용자 요청을 분석하여 개선 대상 파악. 불명확한 경우 질문:

- 어떤 파일/디렉토리/패키지를 개선해야 하는가?
- 특정 관점에 집중할 것인가?

요청이 명확하면 바로 다음 단계로 진행.

### 2. 대상 분석

**목표**: 대상 범위 및 코드 파악

Glob/Grep/Read 도구 또는 Task 도구(subagent_type=Explore)로 대상 분석:

- 대상 유형 판별 (파일/디렉토리/패키지/프로젝트)
- 코드 구조 파악
- 기존 패턴 확인

### 3. 개선 관점 선택

**목표**: 사용자에게 개선 관점 확인

대상 범위에 따른 **권장 관점 제시**:

| 대상   | 권장 관점            |
|------|------------------|
| 파일   | 코딩규칙, 구현품질       |
| 디렉토리 | 코딩규칙, 구현품질, 아키텍처 |
| 패키지  | 전체               |
| 프로젝트 | 전체               |

AskUserQuestion으로 사용자에게 확인 (복수 선택 가능). 응답 없거나 "전부"/"다해줘" 요청 시 권장 관점 전체 적용.

**개선 관점 (에이전트)**:

| 관점   | 에이전트                    | 검토 항목                   |
|------|-------------------------|-------------------------|
| 아키텍처 | sd-improve-architecture | 구조, 의존성, 패턴, 확장성, 계층    |
| 코딩규칙 | sd-improve-coding       | 네이밍, 타입, ESLint, 형식, 주석 |
| 구현품질 | sd-improve-impl         | 에러처리, 테스트, 성능, 보안       |

### 4. 에이전트 호출

**목표**: 선택된 관점의 에이전트 병렬 호출

Task 도구로 선택된 에이전트들을 **병렬 호출**:

- `subagent_type`: `sd-improve-architecture`, `sd-improve-coding`, `sd-improve-impl`
- 프롬프트:

```
다음 대상을 분석하고 이슈를 보고하라:
대상: <경로>
```

### 5. 결과 취합

**목표**: 에이전트별 결과 통합 및 관점 태깅

취합 규칙:

1. 에이전트별 결과를 심각도별로 분류
2. **동일 위치 + 동일 유형**의 이슈만 중복 제거 (심각도 높은 것 우선)
3. 다른 관점의 이슈는 동일 위치라도 모두 유지
4. 심각도 순서로 정렬: Critical → Warning → Info
5. 각 이슈에 출처 관점 태그 추가

### 6. 타당성 검토

**목표**: 오탐(false positive) 필터링

취합된 결과에서 **버그성 문제만 필터링**하여 제외:

| 제외 대상  | 설명                |
|--------|-------------------|
| 사실 오류  | 지적 내용이 사실과 다름     |
| 잘못된 참조 | 존재하지 않는 파일/위치 참조  |
| 가이드 오해 | 가이드를 잘못 해석함       |
| 이미 해결됨 | 지적 사항이 이미 반영되어 있음 |
| 범위 벗어남 | 대상과 무관한 지적, 신규 기능 |

**주의사항:**

- Breaking Change나 테스트 수정이 필요하다는 이유로 제외하지 않음. 더 좋은 방향으로의 수정이라면 "Breaking Change나 테스트 수정"이 불가피한 것으로
  봄.
- 플랜모드에서 레포팅시 제외대상은 표시하지 않음.

### 7. 개선 실행

**목표**: 유효한 이슈들 자동 수정

- 취합된 이슈들을 **자동으로 수정**
- 수정 순서: 파일별 → 라인번호 역순 (뒤에서부터)
- 각 수정 후 즉시 다음 수정 진행

### 8. 코드 검증

**목표**: 타입체크 + 린트 + 테스트

실패 시 수정 후 재검증.

### 9. 최종 결과

**목표**: 개선 결과 제시

```markdown
## 코드개선 결과

**대상**: <경로>
**개선 관점**: <선택된 관점들>

### 개선 항목

#### 🔴 심각 (Critical) - N건 수정됨

**1. <문제 제목> (`파일명:라인번호`)** [<관점>]

- **문제**: <이전 상태>
- **수정**: <변경 내용>

#### 🟡 권장 (Warning) - N건 수정됨

...

#### 🔵 개선 (Info) - N건 수정됨

...

### 검증 결과

- 타입체크: ✅ 통과
- 린트: ✅ 통과
- 테스트: ✅ 통과

### 요약

- 총 개선: N건
- 수정 파일: N개
```
