# 에이전트 소개 및 작성법

특정 역할/전문성을 가진 자율 전문가 (Task 도구로 호출)

## 에이전트가 제공하는 것

1. 성격
2. 태도
3. 가치관

## 에이전트 구조

```
.claude/agents/
└── <agent-name>.md (필수)
    ├── YAML Frontmatter (name, description 필수)
    └── 마크다운 지침
```

## <agent-name>.md 작성법

### Frontmatter

#### name (필수)

에이전트 식별자.

#### description (필수)

에이전트가 언제 사용되어야 하는지 설명하는 핵심 필드.

**참고:** 에이전트는 스킬에서 Task 도구로 직접 호출되므로 복잡한 트리거 패턴이 불필요.

#### color (선택)

콘솔창에 표시될 색상.
기본값은 생략이며, AI가 적어넣지는 않음.
(사용자가 필요에 의해 자유롭게 입력)

#### model (선택)

AI가 사용할 모델.
기본값은 생략이며, AI가 적어넣지는 않음.
(사용자가 필요에 의해 자유롭게 입력)

### 마크다운 본문

#### 구성 요소

| 요소      | 설명                    | 예시                               |
|---------|-----------------------|----------------------------------|
| 역할 정의   | 전문성/성격/태도가 포함된 역할     | "깊이 이해하고 확신 있는 결정을 내리는 시니어 아키텍트" |
| 핵심 원칙   | 달성해야 할 구체적이고 깊이 있는 목표 | "완벽하게 이해할 수 있도록 분석 제공"           |
| 입력      | 에이전트가 받아야 할 정보        | "가이드 파일", "검토 대상 파일"             |
| 관점      | 분석/검토 시 적용할 기준        | "가이드 준수", "품질"                   |
| 접근 방식   | 어떻게 일하는지, 무엇을 중요시하는지  | "기존 패턴 파악 → 명확한 결정 → 실행 가능한 청사진" |
| 산출물 가이드 | 결과물의 특성과 품질 기준        | "구체적이고 실행 가능하게, 파일:라인 포함"        |

#### 작성 원칙

**전문가로서의 성격 부여:**

- 단순 역할이 아닌, 특정 전문성/태도를 가진 사람으로 정의
- 나쁜 예: "당신은 코드 리뷰어입니다"
- 좋은 예: "당신은 깊이 있는 분석과 명확한 피드백을 제공하는 시니어 코드 리뷰어입니다"

**구체적 임무와 가치:**

- "책임"보다 "임무" (더 깊이 있는 목표)
- 무엇을 중요시하는지 명시 (예: "완벽하게 이해", "명확한 결정", "실행 가능한")

**일하는 방식 명시:**

- "프로세스"보다 "접근 방식" (어떻게 일하는지, 무엇을 중요시하는지)
- 원칙 추가 (예: "여러 옵션보다 하나의 확실한 선택")

**작성 시 주의사항:**

| 항목  | 나쁜 예           | 좋은 예                            |
|-----|----------------|---------------------------------|
| 인칭  | "나는 분석합니다"     | "당신은 분석합니다"                     |
| 역할  | "당신은 분석 도구입니다" | "당신은 깊이 있는 분석을 제공하는 시니어 분석가입니다" |
| 표현  | "코드를 확인합니다"    | "타입 안전성과 에러 처리를 확인합니다"          |
| 단계명 | "1단계, 2단계"     | "패턴 파악", "명확한 결정"               |

길이: 10,000단어 초과 지양.
