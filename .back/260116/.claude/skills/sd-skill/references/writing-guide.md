# 스킬 소개

스킬은 전문 지식, 워크플로우를 제공하여 Claude Code의 기능을 확장하는 모듈형 패키지입니다.

## 핵심 원칙

**기본가정: 클로드는 이미 매우 똑똑합니다.** 클로드가 모르는 내용만 추가하여 최대한 간결하게 작성합니다.

## 스킬 구조

```
skill-name/
├── SKILL.md (필수)
│   ├── YAML Frontmatter (name, description 필수)
│   └── 본문
└── 번들 리소스 (선택)
    ├── scripts/
    ├── references/
    └── assets/
```

## 점진적 로딩

컨텍스트를 효율적으로 관리하기 위해 3단계 로딩을 지원합니다.

1. Frontmatter: 항상 로드
2. SKILL.md 본문: 트리거 시 로드
3. 번들 리소스: 필요 시 로드

## SKILL.md 작성법

### Frontmatter

- `name`과 `description`으로 클로드가 스킬의 사용 시점을 결정합니다.
- `description`에는 스킬의 사용 시점을 구체적으로 작성합니다.
- `description`은 3인칭 표현을 사용합니다. (예: 이 스킬은 ...를 요청할 때 사용해야 합니다.)

**예시:**

```yaml
---
name: skill-name
description: 이 스킬은 사용자가 "특정 문구 1", "특정 문구 2"를 요청할 때 사용해야 합니다.
---
```

### 마크다운 본문

**구성 요소:**

- 핵심 개념 및 원칙
- 가장 일반적인 사용 사례
- 워크플로우

**작성 원칙:**

- 명사형/간결체(명령형) 형태 사용 (2인칭 금지)
- 무엇을 해야 하는지에 집중 (예: "A를 수행하려면 B를 한다")
- 특정 상황에서만 필요한 내용은 references/로 분리 (필수)
- markdown 템플릿의 placeholder의 형식은 `<placeholder>`로 작성

**리소스 참조 규칙:**

경로에 스킬명을 포함하여 명확성 확보:

| 참조 형식                           | 용도             |
|---------------------------------|----------------|
| `skill-name/references/file.md` | SKILL.md 본문 내  |
| `.claude/skills/skill-name/...` | Task/에이전트 프롬프트 |
| `.claude/references/file.md`    | 프로젝트 공유 레퍼런스  |

## 번들 리소스

### scripts/

결정론적 신뢰성이 필요하거나 반복 수행되는 작업을 위한 실행 가능한 스크립트.

| 항목    | 설명                          |
|-------|-----------------------------|
| 포함 시점 | 동일 작업 반복, 결정론적 신뢰성 필요 시     |
| 예시    | `scripts/validate-agent.sh` |
| 장점    | 토큰 효율적, 결정론적, 컨텍스트 로드 없이 실행 |

**작성 규칙:**

```bash
#!/bin/bash
# 스크립트 설명 (한 줄)
set -euo pipefail

if [ $# -eq 0 ]; then
  echo "사용법: $0 <args>"
  exit 1
fi

# 주요 로직
echo "✅ 성공" / echo "❌ 실패"
```

- shebang + `set -euo pipefail` 필수
- 인자 없으면 사용법 출력
- 결과는 이모지로 표시 (✅/❌/⚠️)

### references/

SKILL.md와 분리된 문서 및 참조 자료.

| 항목    | 설명                                                        |
|-------|-----------------------------------------------------------|
| 필수 분리 | 특정 상황에서만 필요한 내용 (예: `troubleshooting.md`, `migration.md`) |
| 선택 분리 | 분량이 크거나 독립적 관리가 필요한 문서 (예: `schema-guide.md`)             |
| 장점    | 컨텍스트 효율적, 유지보수성 향상                                        |
| 중복 방지 | SKILL.md 또는 레퍼런스 파일 중 한 곳에만 작성                             |

**작성 규칙:**

- 파일명은 케밥 케이스로 용도를 명확히 표현
- SKILL.md와 동일한 스타일 (명사형/간결체(명령형), 2인칭 금지)
- `# 제목`으로 시작, 제목 섹션 내에 문서 요약 작성
- 테이블, 코드 블록 적극 활용
- SKILL.md와 내용 중복 금지

### assets/

출력에 바로 사용되는 파일. 컨텍스트에 로드하지 않고 복사/포함.

| 항목    | 설명                  |
|-------|---------------------|
| 포함 시점 | 최종 출력에 포함시킬 파일 필요 시 |
| 사용 사례 | 이미지, 아이콘, 폰트, 바이너리  |
| 장점    | 출력 리소스와 문서 분리       |

**관리 규칙:**

- 파일명은 용도를 명확히 표현 (예: `logo-dark.png`)
- 필요시 하위 디렉토리로 분류 (예: `assets/icons/`)

---

# <스킬 제목>

<스킬이 무엇을 제공하는지>

- <질문사항>

요청 명확 시 다음 단계 진행.

### 1. 가이드 분석

- <가이드명>: `<레퍼런스 경로>`
- <가이드명>: `<레퍼런스 경로>`

### 템플릿 placeholder 설명

| Placeholder | 설명 | 예시 |
|-------------|------|------|
| `<skill-name>` | 스킬 디렉토리명 (케밥 케이스) | `sd-dev`, `code-review` |
| `<트리거 문구 N>` | 사용자가 스킬을 호출할 때 사용하는 문구 | "코드 리뷰", "기능 개발" |
| `<사용 상황>` | 스킬이 자동으로 활성화되어야 하는 상황 | "PR 생성 시", "버그 수정 요청 시" |
| `<스킬 제목>` | 스킬의 목적을 나타내는 제목 | "코드 리뷰 자동화" |
| `<단계명>` | 워크플로우 단계의 이름 | "계획 수립", "검증", "실행" |
| `<목표>` | 해당 단계에서 달성할 목표 | "작성 계획 준비", "지침 준수 확인" |
| `<가이드명>` | 참고할 가이드 문서의 이름 | "작성 가이드", "검토 형식" |
| `<레퍼런스 경로>` | 레퍼런스 파일의 상대 경로 | `skill-name/references/guide.md` |
