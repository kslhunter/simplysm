# 프로젝트를 위한 스킬 개발

프로젝트의 `.claude/skills/` 디렉토리에 "프로젝트에 도움이 되는 스킬"을 작성하는 지침 제공.

## 워크플로우

| 단계 | 목표 | 작성/수정 | 검토만 |
|------|------|----------|--------|
| 0. 요청 이해 | 작업 유형 파악 | ✓ | ✓ |
| 1. 가이드 분석 | 작성 기준 파악 | ✓ | ✓ |
| 2. 계획 수립 | 작성 계획 준비 | ✓ | - |
| 3. 검증 | 지침 준수 확인 | 계획 검증 | 대상 파일 검증 |
| 4. 타당성 검토 | 버그성 오류 필터링 | ✓ | ✓ |
| 5. 실행 | 파일 작성/수정 | ✓ | - |
| 6. 최종 결과 | 결과 제시 | 작업요약 | 종합 검토결과 |

### 0. 요청 이해

사용자 요청을 분석하여 **작업 유형** 파악:
- **작성**: 새 스킬 생성
- **수정**: 기존 스킬 변경
- **검토**: 기존 스킬 검증만 수행

불명확한 경우 질문:
- 스킬의 목적은?
- 대상 독자는?
- 언제 사용해야 하나?
- 사용자 트리거 문구는?

요청 명확 시 다음 단계 진행.

### 1. 가이드 분석

- 작성 가이드: `sd-skill/references/writing-guide.md`
- 검토 형식: `.claude/references/review-format.md`
- (수정/검토 시) 대상 스킬 파일 분석

### 2. 계획 수립

다음 항목을 포함한 구체적 계획 준비:
- 스킬 디렉토리 구조 (SKILL.md, references/, scripts/ 필요 여부)
- Frontmatter 내용 (name, description)
- SKILL.md 주요 섹션 구성
- references/ 파일 목록 및 내용 개요
- scripts/ 필요 여부 및 기능 목록

계획 준비 후 간략히 제시.

### 3. 검증

**작성/수정 시**: 2단계 계획을 검증
**검토 시**: 대상 스킬 파일을 검증

Task 도구로 `sd-instruction-reviewer` 에이전트 호출:
```
레퍼런스: .claude/skills/sd-skill/references/writing-guide.md
검토 대상: <계획 내용 또는 스킬 디렉토리 경로>
```

예시:
```
레퍼런스: .claude/skills/sd-skill/references/writing-guide.md
검토 대상: .claude/skills/sd-dev
```

#### 3.1. 레퍼런스 검토 (검토 시)

대상 스킬이 사용하는 레퍼런스 파일 품질 검증.

대상 스킬이 사용하는 레퍼런스가 있다면 Task 도구로 `sd-instruction-reviewer` 에이전트 호출:
```
레퍼런스: .claude/skills/sd-skill/references/writing-guide.md
추가 지침: "references/" 섹션 기준으로 검토
검토 대상: 검토 대상 스킬이 사용하는 레퍼런스 파일들
```

#### 3.2. 에이전트 검토 (검토 시)

대상 스킬이 사용하는 에이전트 파일 품질 검증.

대상 스킬이 사용하는 에이전트가 있다면 에이전트 내용과 스킬 내용 사이의 연계가 적절한지 직접 검토:
- 호출 형식, 입력/출력 매칭, 결과 활용 관점

### 4. 타당성 검토 (**중요**)

3단계 검증 결과에서 아래 **버그성 문제**를 직접 각 관련파일을 세밀히 검토하여 제외:

- **사실 오류**: 지적 내용이 사실과 다름 (예: "A가 없다"고 지적했는데 실제로 A가 존재함)
- **잘못된 참조**: 존재하지 않는 파일/위치 참조
- **가이드 오해**: 가이드를 잘못 해석함 (예: 선택적 항목을 필수로 오해)
- **이미 해결됨**: 지적 사항이 이미 반영되어 있음
- **범위 벗어남**: 검증 대상과 무관한 지적

위 항목에 해당하는 버그성 지적만 제외하고, 나머지 스타일/표현/구조 개선 제안은 유효한 피드백으로 유지.

**결과물 활용**:
- **작성/수정 시**: 유효한 피드백을 2단계 계획에 반영하여 계획 업데이트
- **검토 시**: 유효한 피드백을 6단계 검토 결과로 제시

### 5. 실행

4단계에서 업데이트된 계획을 기반으로 실제 파일 작성/수정.

권장 작성 순서:
1. SKILL.md 작성 (frontmatter + 본문)
2. references/ 파일 작성 (SKILL.md에서 분리할 상세 내용)
3. scripts/ 작성 (필요시)

### 6. 최종 결과

**작성/수정 시 - 작업 요약**:
- 생성/수정된 파일 목록
- 주요 작성 내용
- 적용된 구조 및 패턴

**검토 시 - 검토 결과**:
- 3단계(SKILL.md 검토, 레퍼런스 검토, 에이전트 검토) 결과를 통합하여 4단계에서 필터링된 최종 개선 항목 제시
