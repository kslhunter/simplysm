# 번역 Skill

코드 주석, UI 텍스트, 문서, 일반 텍스트 등 다양한 컨텍스트에서 다국어 번역 수행.

## 핵심 원칙

- **직역보다 의역**: 문맥과 의도를 살린 자연스러운 표현 선택
- **형식 유지**: 코드 구조, 마크다운, JSON 등 원본 형식 보존
- **기술 용어 유지**: 도메인/기술 용어는 원문 그대로
- **일관성 유지**: 동일 용어는 문서 전체에서 동일하게 번역

## 워크플로우

| 단계 | 목표 |
|------|------|
| 0. 요청 이해 | 핵심 목표 파악 |
| 1. 원문 확인 | 번역 대상 파악 |
| 2. 언어 감지 | 원문/대상 언어 결정 |
| 3. 번역 수행 | 형식 유지하며 번역 |
| 4. 검토 | 품질 확인 |
| 5. 결과 제시 | 번역 결과 전달 |

### 0. 요청 이해

사용자 요청을 분석하여 핵심 목표 파악. 불명확한 경우 질문:
- 번역할 텍스트/파일 범위는?
- 대상 언어는?

요청이 명확하면 바로 다음 단계로 진행.

### 1. 원문 확인

번역 대상 텍스트 또는 파일 확인:
- 텍스트: 사용자가 제공한 내용 그대로 사용
- 파일: Read 도구로 파일 내용 읽기, 형식(마크다운, JSON, 코드 등) 파악

### 2. 언어 감지

원문 언어 자동 감지 또는 사용자 지정 언어 사용.
대상 언어 미지정 시 사용자에게 확인.

### 3. 번역 수행

핵심 원칙에 따라 번역:
- 문맥과 의도를 살린 자연스러운 표현 선택
- 코드 구조, 마크다운, JSON 등 원본 형식 보존
- 도메인/기술 용어는 원문 그대로 유지
- 동일 용어는 문서 전체에서 일관되게 번역

### 4. 검토

번역 결과 품질 확인:
- 형식 보존: 원본과 동일한 구조인가?
- 용어 일관성: 동일 용어가 일관되게 번역되었는가?
- 자연스러움: 목표 언어 화자가 읽기 자연스러운가?

### 5. 결과 제시

번역 결과 파일로 저장:
- 기존 파일: Edit 도구로 해당 부분 수정
- 신규 파일: Write 도구로 저장 (경로는 사용자에게 확인)
