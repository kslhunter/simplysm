# Requirements Specification — UI / Component

Add the following sections when the task involves creating or modifying a user interface element.

## Component Hierarchy

Describe the visual structure. List parent-child relationships between components.

## States and Behavior

For each component state:

| Item | Description |
|------|-------------|
| **State** | Name (e.g., default, loading, error, empty, disabled) |
| **Visual** | How it looks in this state |
| **Behavior** | What interactions are available |
| **Transition** | What causes transition to/from this state |

## User Interaction Flow

Describe the step-by-step user journey:

1. User sees/does X
2. System responds with Y
