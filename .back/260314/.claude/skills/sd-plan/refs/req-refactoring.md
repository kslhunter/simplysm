# Requirements Specification — Refactoring

Add the following sections when the task involves restructuring code without changing external behavior.

## Current State

Describe the current code structure and why it needs refactoring.

## Target State

Describe the desired code structure after refactoring.

## Behavior Preservation

Explicitly list behaviors/APIs that MUST remain unchanged.

## Migration (if applicable)

If the refactoring involves breaking changes to internal APIs:
- What needs to be updated
- Migration strategy (big bang vs. gradual)
