# Requirements Specification — Common Sections

Include ALL of the following sections in every requirements specification, regardless of type.

## Goal

State WHY this task is being done in 1-2 sentences. Focus on the problem being solved or value delivered.

## Scope

- **IN scope**: Explicit list of what is included
- **OUT of scope**: Explicit list of what is excluded

## Feature List

For each feature:

| Item | Description |
|------|-------------|
| **Name** | Short descriptive name |
| **Description** | What the feature does |
| **Input** | Data/action that triggers this feature |
| **Output** | What the feature produces or changes |
| **Exceptions** | Error conditions and expected handling |

## Constraints (include only if applicable)

Non-functional requirements: performance, compatibility, platform, security, etc.
