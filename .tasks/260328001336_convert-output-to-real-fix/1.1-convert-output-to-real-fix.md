# Feature 1.1 _convertOutputToReal dist/src/ 패턴 보정

## 참조 자료

### 근본 원인

- `SdTsCompiler._convertOutputToReal()` (`packages/sd-cli/src/ts-compiler/SdTsCompiler.ts:597-615`)이 `dist/<패키지명>/src/` 패턴만 보정한다
- TypeScript의 `commonSourceDirectory` 추론에 따라 출력 경로가 달라진다:
  - test 파일이 `@simplysm/*`를 import하면 → `commonSourceDirectory = packages/` → 출력: `dist/<패키지명>/src/` → 기존 보정 성공
  - test 파일이 외부 패키지를 import하지 않으면 → `commonSourceDirectory = 패키지 루트` → 출력: `dist/src/` → 보정 실패
- 실제 영향 패키지: `capacitor-plugin-intent` (test 파일이 `@simplysm/*` 미사용)

### 기존 보정 로직 (현행)

```typescript
const srcRelBasePath = path.resolve(distPath, path.basename(this._opt.pkgPath), "src");
// dist/<패키지명>/src/ 패턴만 매칭
if (PathUtils.isChildPath(realFilePath, srcRelBasePath)) {
  realFilePath = PathUtils.norm(distPath, path.relative(srcRelBasePath, realFilePath));
}
```

## 구현계획

### 배경

`SdTsCompiler._convertOutputToReal()`은 TypeScript emit 결과의 출력 경로를 후처리로 보정한다. TypeScript의 `commonSourceDirectory` 추론에 따라 출력 경로가 `dist/<패키지명>/src/` 또는 `dist/src/`가 될 수 있으나, 현재 전자만 처리한다.

### 목표

- `dist/src/` 패턴도 `dist/`로 보정
- source map의 sources 경로도 패턴에 맞게 동적 보정
- 기존 `dist/<패키지명>/src/` 패턴 동작 유지

### 비목표

- tsconfig.json의 exclude 수정 (패키지별 대응이므로 근본 해결 아님)
- `commonSourceDirectory` 추론 자체를 변경

### 설계

`_convertOutputToReal`에서 두 패턴을 순차 매칭:
1. `dist/<패키지명>/src/` (기존 패턴)
2. `dist/src/` (신규 패턴)

source map 보정은 하드코딩된 `.slice(6)` 대신 실제 경로 이동 depth에서 동적으로 계산한다. `filePath`와 `realFilePath`의 directory depth 차이로 `"../"` 반복 횟수를 구하고, 그만큼 sources 경로에서 prefix를 제거한다.

### 대안 검토

| 접근 방식 | 선택 여부 | 이유 |
|-----------|-----------|------|
| 두 패턴을 순차 매칭 | 채택 | 기존 로직과의 호환성 유지, 변경 최소화 |
| `rootDir: "./src"`를 tsconfig에 강제 | 미채택 | sd-cli가 tsconfig를 수정하지 않는 설계 원칙 위반 |
| tests를 exclude에 추가 | 미채택 | 각 패키지별 대응 필요, 근본 해결 아님 |
