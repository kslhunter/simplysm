# Feature 1.1 패키지 순차 배포

## 참조 자료

### 근본 원인 (debug.md)
- `SdCliProject.publishAsync`에서 `parallelAsync`(`Promise.all`)로 21개 `yarn npm publish`를 동시 실행하면 Yarn 내부 락 경합으로 모든 프로세스가 블로킹됨
- 관련 파일: `packages/sd-cli/src/entry/SdCliProject.ts:336`

### 현재 구현
- `parallelAsync`는 `Promise.all`로 구현 (`packages/sd-core-common/src/extensions/Array.ext.ts:246`)
- `_publishPkgAsync`는 `SdProcess.spawnAsync`로 자식 프로세스 생성 (`SdCliProject.ts:375-427`)

## 요구명세

Feature: 1.1 패키지 순차 배포

  Background:
    Given sd-cli의 publishAsync가 배포 단계에 진입했다
    And 배포 대상 패키지가 1개 이상 존재한다

  Rule: 패키지 배포는 순차적으로 실행한다

    Scenario: 모든 패키지 순차 배포 성공
      Given 21개 패키지가 배포 대상이다
      When 배포를 실행한다
      Then 각 패키지가 한 번에 하나씩 순서대로 배포된다
      And 모든 패키지의 "배포 완료" 로그가 출력된다

    Scenario: 중간 패키지 배포 실패 시 중단
      Given 3번째 패키지의 배포가 실패한다
      When 배포를 실행한다
      Then 1, 2번째 패키지는 배포 완료된다
      And 3번째 패키지에서 에러가 throw된다
      And 4번째 이후 패키지는 배포가 시작되지 않는다

  Rule: publish 설정이 없는 패키지는 건너뛴다

    Scenario: publish 미설정 패키지 건너뛰기
      Given 일부 패키지의 publish 설정이 null이다
      When 배포를 실행한다
      Then 해당 패키지는 건너뛰고 다음 패키지로 진행한다

## 구현계획

### 배경

`SdCliProject.publishAsync`의 배포 루프가 `parallelAsync`(`Promise.all`)로 21개 `yarn npm publish`를 동시 실행하여 Yarn 내부 락 경합으로 프로세스가 멈춘다.

### 목표

- 배포 루프를 순차 실행으로 변경하여 Yarn 락 경합을 제거한다

### 비목표

- concurrency limiter 도입 (단순 순차 실행으로 충분)
- `parallelAsync` 유틸리티 자체 수정 (다른 곳에서 정상 사용 중)

### 설계

`SdCliProject.ts:336`의 `pkgPaths.parallelAsync(async (pkgPath) => { ... })` 를 `for...of` 루프로 교체한다.

변경 포인트:
- `parallelAsync` 콜백 → `for...of` 루프
- 콜백 내 `return` (skip) → `continue`

### 대안 검토

| 접근 방식 | 선택 여부 | 이유 |
|-----------|-----------|------|
| `for...of` 순차 실행 | 채택 | 가장 단순, 락 경합 완전 제거 |
| 배치(batch) 병렬 실행 | 미채택 | Yarn 프로젝트 레벨 락 경합 가능성 잔존 |
