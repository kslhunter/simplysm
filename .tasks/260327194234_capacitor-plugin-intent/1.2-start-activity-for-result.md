# Feature 1.2 startActivityForResult API 추가

## 참조 자료

- [wbs.md](./wbs.md)

### 설계 결정

| # | 결정사항 | 선택 | 근거 |
|---|---------|------|------|
| D1 | resultCode가 RESULT_CANCELED(0)일 때 처리 | 항상 resolve (resultCode 값으로 판별) | Android API 의미론과 일치, 호출자가 resultCode로 세밀하게 분기 가능, 정상 흐름(뒤로가기)이 에러 경로를 타지 않음 |


## 구현계획

### 배경

`capacitor-plugin-intent`는 Feature 1.1에서 `capacitor-plugin-broadcast`로부터 리네이밍된 Capacitor 플러그인이다. 현재 Broadcast Intent 송수신(subscribe, send) 및 Launch Intent 조회(getLaunchIntent) API를 제공한다. Android 네이티브 레이어(`IntentPlugin.java`)에 `populateExtras`, `intentToJson`, `bundleToJson` 유틸 메서드가 이미 존재하며, 이를 재활용한다.

### 목표

- `startActivityForResult` API를 TypeScript 인터페이스, Android 네이티브, 웹 stub에 추가한다
- Intent 구성 옵션(action, uri, extras, package, component, type)을 지원한다
- Activity 결과를 `{ resultCode, data?, extras? }` 형태로 반환한다

### 비목표

- iOS 지원
- Web 네이티브 구현 (stub만 제공)
- flags, categories 등 사용 빈도 낮은 Intent 속성

### 설계

#### TypeScript 타입

```typescript
// IIntentPlugin.ts에 추가
interface IStartActivityForResultOptions {
  action: string;
  uri?: string;
  extras?: Record<string, unknown>;
  package?: string;
  component?: string;
  type?: string;
}

interface IActivityResult {
  resultCode: number;
  data?: string;
  extras?: Record<string, unknown>;
}
```

`IIntentPlugin`에 메서드 추가:
```typescript
startActivityForResult(options: IStartActivityForResultOptions): Promise<IActivityResult>;
```

#### Android 네이티브 (`IntentPlugin.java`)

- `@PluginMethod` `startActivityForResult(PluginCall call)`:
  1. `call`에서 options 추출 (action, uri, extras, package, component, type)
  2. `Intent` 객체 구성:
     - `new Intent(action)`
     - uri만 있으면 `intent.setData(Uri.parse(uri))`
     - type만 있으면 `intent.setType(type)`
     - uri + type 동시 → `intent.setDataAndType(Uri.parse(uri), type)`
     - package + component → `intent.setComponent(new ComponentName(package, component))`
     - extras → 기존 `populateExtras(intent, extras)` 재활용
  3. `startActivityForResult(call, intent, "onActivityResult")` 호출
- `@ActivityCallback` `onActivityResult(PluginCall call, ActivityResult result)`:
  1. `result.getResultCode()` → `resultCode`
  2. `result.getData()` (Intent) → `data`: `intent.getDataString()`, `extras`: 기존 `bundleToJson(intent.getExtras())` 재활용
  3. JSObject로 조합하여 `call.resolve(ret)` (항상 resolve)

#### TypeScript 래퍼 (`Intent.ts`)

```typescript
static async startActivityForResult(options: IStartActivityForResultOptions): Promise<IActivityResult> {
  return await IntentPlugin.startActivityForResult(options);
}
```

#### 웹 stub (`IntentWeb.ts`)

```typescript
async startActivityForResult(_options: IStartActivityForResultOptions): Promise<IActivityResult> {
  alert("[Intent] 웹 환경에서는 startActivityForResult를 지원하지 않습니다.");
  return await Promise.resolve({ resultCode: 0 });
}
```

### 대안 검토

| 접근 방식 | 선택 여부 | 이유 |
|-----------|-----------|------|
| Capacitor `startActivityForResult` + `@ActivityCallback` | 채택 | Capacitor Plugin API의 공식 패턴, 자동 request code 관리 |
| 직접 `Activity.startActivityForResult` + `onActivityResult` 오버라이드 | 미채택 | request code 수동 관리 필요, Capacitor Plugin 라이프사이클과 충돌 가능 |
