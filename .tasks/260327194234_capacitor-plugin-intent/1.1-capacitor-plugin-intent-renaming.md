```gherkin

      And packages/capacitor-plugin-broadcast/ 경로는 존재하지 않는다

      When simplysm.js의 패키지 키를 capacitor-plugin-intent로 변경한다
      Then simplysm.js에서 capacitor-plugin-intent 키로 패키지가 등록되어 있다
      And capacitor-plugin-broadcast 키는 존재하지 않는다

      Then IntentWeb.ts에 class IntentWeb extends WebPlugin implements IIntentPlugin이 존재한다

      When index.ts의 export 경로를 변경한다

      Given Intent 플러그인이 로드되어 있다
      When Intent.subscribe(["com.example.ACTION"], callback)을 호출한다
      Then 브로드캐스트 리시버가 등록되고 unsubscribe 함수가 반환된다

      When 루트 README.md의 패키지 목록을 갱신한다
      Then capacitor-plugin-intent 항목이 존재한다
      And capacitor-plugin-broadcast 항목은 존재하지 않는다

      Then @simplysm/capacitor-plugin-broadcast 설치 시 deprecated 경고가 표시된다
```

## 구현계획

### 배경

monorepo의 `packages/capacitor-plugin-broadcast/` 패키지를 `capacitor-plugin-intent`로 리네이밍한다. 이 패키지는 Android Broadcast 송수신을 위한 Capacitor 플러그인으로, Feature 1.2에서 `startActivityForResult` API를 추가할 예정이므로 "Broadcast"보다 "Intent"가 더 포괄적인 이름이다.

변경 대상 파일:
- `packages/capacitor-plugin-broadcast/` (디렉토리 전체)
- `packages/capacitor-plugin-broadcast/package.json` (name, description, repository.directory)
- `packages/capacitor-plugin-broadcast/android/build.gradle` (namespace)
- `packages/capacitor-plugin-broadcast/android/src/main/java/kr/co/simplysm/capacitor/broadcast/BroadcastPlugin.java` (파일 경로, 패키지, 클래스명, 어노테이션)
- `packages/capacitor-plugin-broadcast/src/Broadcast.ts` → `Intent.ts`
- `packages/capacitor-plugin-broadcast/src/IBroadcastPlugin.ts` → `IIntentPlugin.ts`
- `packages/capacitor-plugin-broadcast/src/web/BroadcastWeb.ts` → `IntentWeb.ts`
- `packages/capacitor-plugin-broadcast/src/index.ts` (export 경로)
- `packages/capacitor-plugin-broadcast/README.md`
- `simplysm.js` (패키지 키)
- `README.md` (루트, 패키지 목록)

monorepo 내 다른 패키지에서 이 플러그인을 import하는 곳은 없다. `tsconfig.base.json`에 경로 별칭도 없다.

### 목표

- 모든 "broadcast/Broadcast" 참조를 "intent/Intent"로 일괄 변경
- 기존 API(subscribe, unsubscribe, unsubscribeAll, send, getLaunchIntent) 시그니처 및 동작 보존
- TypeScript 컴파일 성공 확인

### 비목표

- API 로직 변경 (순수 리네이밍만 수행)
- startActivityForResult 추가 (Feature 1.2)
- iOS 지원

### 설계

순수 리네이밍이므로 구조적 설계 변경 없음. 변경 매핑:

| 대상 | Before | After |
|------|--------|-------|
| 디렉토리 | `packages/capacitor-plugin-broadcast/` | `packages/capacitor-plugin-intent/` |
| npm name | `@simplysm/capacitor-plugin-broadcast` | `@simplysm/capacitor-plugin-intent` |
| Java package | `kr.co.simplysm.capacitor.broadcast` | `kr.co.simplysm.capacitor.intent` |
| Java class | `BroadcastPlugin` | `IntentPlugin` |
| Capacitor name | `"Broadcast"` | `"Intent"` |
| TS class | `Broadcast` | `Intent` |
| TS interfaces | `IBroadcastPlugin`, `IBroadcastResult` | `IIntentPlugin`, `IIntentResult` |
| TS web class | `BroadcastWeb` | `IntentWeb` |
| gradle namespace | `kr.co.simplysm.capacitor.broadcast` | `kr.co.simplysm.capacitor.intent` |
| simplysm.js key | `"capacitor-plugin-broadcast"` | `"capacitor-plugin-intent"` |
