# Feature 1.1 eslint-plugin rootDir resolve 버그 수정

## 참조 자료

### 버그 증상
- 에러: `ENOENT: no such file or directory, scandir 'D:\workspaces-12\oscom\node_modules\packages'`
- 원인 파일: `packages/eslint-plugin/src/configs/root.js:11`
- `rootDir`이 `import.meta.url` + `"../../../.."`로 계산되어, `node_modules/` 내부 설치 시 소비자 프로젝트 루트가 아닌 `node_modules/`를 가리킴

### 디버그 분석
- [debug.md](./debug.md)

### 설계 결정

| # | 결정사항 | 선택 | 근거 |
|---|---------|------|------|
| D1 | rootDir 계산 방식 | `process.cwd()` | API 브레이킹 체인지 없이 해결. ESLint는 관례적으로 프로젝트 루트에서 실행 |

## 구현계획

### 배경

`@simplysm/eslint-plugin`의 `configs/root.js`는 monorepo의 `packages/` 디렉토리를 스캔하여 `import/no-extraneous-dependencies` 룰의 `packageDir`을 구성한다. `rootDir` 계산이 `import.meta.url` 기반 상대 경로(`"../../../.."`)로 하드코딩되어 있어, 외부 프로젝트에서 `node_modules/`를 통해 설치하면 경로가 깨진다.

### 목표

- `rootDir`이 설치 위치와 무관하게 소비자 프로젝트의 루트를 가리키도록 수정

### 비목표

- `configs.root`의 API 변경 (함수화 등)
- `packages/` 디렉토리 미존재 시 graceful 처리 (기존 동작 유지)

### 설계

`rootDir` 계산을 `process.cwd()`로 대체한다. ESLint는 프로젝트 루트에서 실행되는 것이 관례이므로 `process.cwd()`가 프로젝트 루트를 가리킨다.

변경 파일: `packages/eslint-plugin/src/configs/root.js`
- L11: `const rootDir = path.resolve(path.dirname(fileURLToPath(import.meta.url)), "../../../..");` → `const rootDir = process.cwd();`
- L2: 미사용 `import { fileURLToPath } from "node:url";` 제거

### 대안 검토

| 접근 방식 | 선택 여부 | 이유 |
|-----------|-----------|------|
| `process.cwd()` 사용 | 채택 | API 변경 없음, 관례적 사용 패턴과 일치 |
| `configs.root`를 함수로 변경 | 미채택 | API 브레이킹 체인지, 모든 소비자 코드 수정 필요 |
