# Feature 1.1 sd.config.ts exclude 옵션

## 참조 자료

### 배경
- Capacitor 프로젝트에서 `jeep-sqlite` 같은 패키지는 Vite의 `optimizeDeps.exclude`에 추가해야 정상 동작한다
- 해당 패키지는 Capacitor/Electron의 `package.json`에도 의존성으로 추가되어야 한다
- 현재 sd-cli에는 이를 설정할 방법이 없다

### 결정 사항
- 옵션 위치: `SdClientPackageConfig` 최상위
- 옵션 이름: `exclude`
- 옵션 타입: `string[]`
- 버전 소스: 클라이언트 패키지의 `dependencies`에서 자동 추출
- Electron: `package.json`에도 동일하게 추가

### 관련 코드
- 타입 정의: `packages/sd-cli/src/sd-config.types.ts` (`SdClientPackageConfig`)
- Vite 설정: `packages/sd-cli/src/utils/vite-config.ts` (`createViteConfig`, `sdScopeWatchPlugin`)
- Capacitor package.json: `packages/sd-cli/src/capacitor/capacitor.ts` (`_setupNpmConf`)
- Electron package.json: `packages/sd-cli/src/electron/electron.ts` (`_setupPackageJson`)

## 구현계획

### 배경

sd-cli의 `SdClientPackageConfig`는 client 타겟 패키지의 빌드/개발 설정을 정의한다. 현재 Vite의 `optimizeDeps.exclude`는 `replaceDeps`(로컬 심링크 패키지)에 대해서만 `sdScopeWatchPlugin`에서 자동 설정된다. Capacitor/Electron에서 사용하는 네이티브 바인딩 패키지(`jeep-sqlite` 등)는 별도로 exclude해야 하지만, 이를 설정할 방법이 없다.

### 목표

- `SdClientPackageConfig`에 `exclude?: string[]` 필드 추가
- exclude 패키지를 Vite `optimizeDeps.exclude`에 반영 (dev/build 모두)
- exclude 패키지를 Capacitor `.capacitor/package.json`의 dependencies에 추가
- exclude 패키지를 Electron `.electron/src/package.json`의 dependencies에 추가

### 비목표

- `optimizeDeps.include` 커스터마이징
- exclude 패키지 버전 직접 지정 (항상 클라이언트 dependencies에서 추출)

### 설계

#### 타입 변경

`SdClientPackageConfig`에 필드 추가:
```typescript
exclude?: string[];  // Vite optimizeDeps.exclude + Capacitor/Electron package.json에 추가할 패키지
```

`ViteConfigOptions`에 필드 추가:
```typescript
exclude?: string[];
```

#### Vite 설정 반영

`createViteConfig()`의 반환값에 `optimizeDeps.exclude` 직접 설정. `sdScopeWatchPlugin`의 `config()` 훅이 반환하는 `optimizeDeps.exclude`와는 Vite의 `mergeConfig`에 의해 배열이 자동 병합됨.

#### Capacitor/Electron 전달 경로

`exclude`는 `SdClientPackageConfig` 최상위에 있고, `Capacitor.create()`/`Electron.create()`는 하위 config(`SdCapacitorConfig`/`SdElectronConfig`)만 받으므로, `exclude`를 별도 파라미터로 전달:

- `Capacitor.create(pkgDir, config.capacitor, config.exclude)` → `_setupNpmConf`에서 plugins과 동일 패턴으로 dependencies 추가
- `Electron.create(pkgDir, config.electron, config.exclude)` → `_setupPackageJson`에서 reinstallDependencies와 동일 패턴으로 dependencies 추가

호출 위치 4곳:
- `BuildOrchestrator.ts:363` (Capacitor build)
- `BuildOrchestrator.ts:388` (Electron build)
- `DevOrchestrator.ts:340` (Capacitor dev)
- `commands/device.ts:90,127` (device 명령)

### 대안 검토

| 접근 방식 | 선택 여부 | 이유 |
|-----------|-----------|------|
| `SdClientPackageConfig` 최상위에 `exclude` | 채택 | capacitor/electron 무관하게 Vite exclude 적용 가능 |
| `SdCapacitorConfig`/`SdElectronConfig` 하위에 각각 `exclude` | 미채택 | 설정 중복, Vite exclude와 package.json 추가가 분리됨 |
| `createViteConfig`에서 직접 `optimizeDeps.exclude` 설정 | 채택 | `sdScopeWatchPlugin`과 Vite `mergeConfig`로 자연스럽게 병합 |
| `sdScopeWatchPlugin` 내부에서 merge | 미채택 | `replaceDeps` 없을 때 plugin 자체가 생성되지 않아 별도 처리 필요 |
