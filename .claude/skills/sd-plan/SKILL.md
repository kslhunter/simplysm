# sd-plan: 구현계획서 작성

참조 문서(spec, audit, debug 등)를 기반으로 TDD 방식의 세부 구현계획서(`plan.md`)를 작성한다.

## 참조 문서 수정 규칙

참조 문서에 누락/오류를 발견하면 AskUserQuestion으로 수정 승인을 받고 최소한으로 수정한다.
수정 범위가 과도하면 해당 문서의 재작성을 안내하고 현재 작업을 중단한다.

## 1. 요구사항 분석

요구사항을 독립적 기능 단위로 작업을 분리하고, 각 작업의 유형을 분류하여 테스트 전략을 결정한다.

- 각 작업은 하나의 독립적인 TDD 사이클로 처리할 수 있는 크기여야 한다
- `.claude/skills/sd-ref-tdd/SKILL.md`를 Read하여 "작업 유형 분류" 테이블과 "테스트 시나리오 작성 원칙" 섹션을 참조한다
- 참조한 내용을 기반으로 각 작업의 테스트 전략을 결정하고 RED/GREEN Phase를 계획한다

## 2. 구현계획서 초안 작성

분석 결과를 바탕으로 구현계획서 초안을 작성한다.

### 불확실한 결정의 처리

초안 작성 시 각 결정 사항을 다음 기준으로 분류한다:

- **참조 문서와 코드베이스에서 유일하게 도출되는 사항** → 그대로 작성
- **여러 선택지가 있거나 참조 문서에서 불분명한 사항** → 결정이 필요함을 명시하고 `[?]` 마커를 붙임 (임의로 결정하지 않는다)

`[?]` 마커 예시:

```markdown
테스트 DB 환경을 결정해야 한다 (in-memory SQLite / Docker MySQL / mock). [?]
```

### 문서 템플릿

```markdown
# 구현계획서: {제목}

**참조 문서:** `{참조 문서 경로}` {R번호}
**대상:** `{대상 파일/패키지 경로}` {신규 작성/수정}
**작업 유형:** {유형} → {테스트 전략 요약}

## 작업 분석

{요구사항을 기능 단위로 분리한 결과}

## TDD 계획

### RED Phase: {테스트 파일명} 작성 및 실패 확인

{테스트 시나리오 목록}

### GREEN Phase: {구현 대상} 작성

{구현 계획 상세}

구현 후 테스트 재실행하여 통과 확인

### Refactor Phase

`/sd-simplify` 하여 코드 리팩터링 → 완료 후 멈추지 말고 즉시 테스트 재실행하여 통과 확인
```

**참조 문서 표기 규칙:**

- 참조 문서 파일 경로로 호출된 경우: `**참조 문서:** {경로} {R번호}` 형식으로 상단에 명시
- 현재 대화의 참조 문서인 경우: `**참조 문서:** 현재 대화의 참조 문서` 로 명시
- 참조 문서 없이 호출된 경우: `**참조 문서:**` 줄 생략

**다중 작업인 경우:**

- 독립적인 작업은 같은 Batch로 묶고, 선행 작업이 필요한 경우 별도 Batch로 분리한다
- `## Batch 1`, `## Batch 2` 순서로 구분하고, 각 Batch 내에 `### 작업 1: {제목}` 형태로 나열한다

### 첨부파일

대화 중 등장한 파일(사용자 첨부, sd-doc-extract 추출 등)이 있으면:
1. 저장 디렉토리의 `attachments/` 폴더에 복사한다
2. 문서 내 관련 위치에 링크한다: `![{파일명}](attachments/{파일명})` 또는 `[{파일명}](attachments/{파일명})`

## 3. 초안 기반 검토 질문

초안은 사용자에게 출력하지 않는다 (최종본이 파일로 저장되므로 대화창 출력은 불필요).

초안에서 `[?]` 마킹된 부분이 있으면 **한번에 1개씩만** 다음 프로세스를 반복한다:

1. 하나의 `[?]` 항목을 인용하며 **자세한 설명을 텍스트로 먼저 출력**한다
   - 초안에 나열된 선택지 각각의 장단점, 트레이드오프, 10점만점의 점수를 설명한다
   - 현재 코드베이스의 맥락을 반영한 설명을 제공한다
2. AskUserQuestion을 **1개만** 호출한다
3. 답변을 반영하여 초안을 수정하고 해당 `[?]` 마커를 제거한다
4. 추가 `[?]`가 있으면 1번으로 돌아간다

`[?]` 마킹이 없으면 질문을 건너뛰고 저장으로 넘어간다.

## 4. 파일 저장

- 참조 문서 파일 기반: 해당 문서와 동일 디렉토리에 저장. 전체면 `plan.md`, 특정 R항목이면 `{R번호}_plan.md` (예: `R2_plan.md`). 서브디렉토리(`{R번호}/spec.md`)도 동일 규칙 적용
- 참조 문서 없이 호출: `.tasks/{yyMMddHHmmss}_{topic}/plan.md` (`date +%y%m%d%H%M%S`, topic은 영문 kebab-case)

## 5. 완료 안내

문서 저장 후 다음을 출력한다:
- 파일 경로
- 다음 단계: `/sd-plan-dev {plan경로}`
