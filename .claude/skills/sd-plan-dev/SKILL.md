# sd-plan-dev: 구현 실행

> [!IMPORTANT]
> **절대 중간에 멈추지 마라.** TDD 사이클(3단계), 자가검토(4단계), 완료 안내(5단계)까지 반드시 끝까지 완수한다. 한 단계가 끝나면 즉시 다음 단계로 진행한다. 중간에 멈추거나 완료된 척 하는 것은 허용되지 않는다.

구현계획서(plan)를 기반으로 Batch 단위로 TDD 사이클을 실행하고, 자가검토를 수행한다.

## 1. 인자 파싱

`$ARGUMENTS`를 분석하여 plan을 결정한다.

### 1-1. plan 파일 경로

`$ARGUMENTS`에 `.md`로 끝나는 경로가 포함된 경우:
- 해당 파일을 Read한다

### 1-2. 인자 없음

`$ARGUMENTS`가 비어있으면:
- 현재 대화 맥락에서 plan 내용을 파악한다
- 대화에도 plan이 없으면 다음을 출력하고 종료한다:
  > 구현계획서가 없습니다. 먼저 `/sd-plan`을 실행하여 계획을 수립하세요.

## 참조 문서 수정 규칙

참조 문서에 누락/오류를 발견하면 AskUserQuestion으로 수정 승인을 받고 최소한으로 수정한다.
수정 범위가 과도하면 해당 문서의 재작성을 안내하고 현재 작업을 중단한다.

## 2. 작업 파악 및 실행 전략

plan에서 Batch 구조를 파악하고 실행 순서를 결정한다.

- plan의 `## Batch N` 구조를 읽어 Batch 목록과 각 Batch 내 작업 목록을 파악한다
- Batch가 없으면 모든 작업을 단일 Batch로 간주한다
- Batch 순서대로 실행한다 (Batch 1 완료 후 Batch 2)
- Batch 내 작업이 여러 개면 병렬 실행이 가능하다

## 3. TDD 사이클 실행

`.claude/skills/sd-ref-tdd/SKILL.md`를 Read하여 TDD 방법론 가이드를 참조한다.

각 Batch를 순서대로 실행한다:

1. Batch 내 각 작업에 대해 sd-ref-tdd 가이드의 TDD 사이클(RED → GREEN → Refactor → Commit)을 수행한다
   - 작업의 유형 분류는 가이드의 "작업 유형 분류" 테이블에 따른다
   - 테스트 시나리오는 가이드의 "테스트 시나리오 작성 원칙"에 따른다
   - 각 Phase는 가이드의 RED/GREEN/Refactor/Commit Phase 절차에 따른다
2. Batch 내 모든 작업이 완료되면 다음 Batch로 진행한다
3. 모든 Batch가 완료되면 4단계로 진행한다

## 4. 자가검토

모든 작업의 TDD 사이클이 완료되면 plan 원문과 실제 구현 결과를 대조한다.

1. plan 원문을 다시 읽는다
2. 각 작업별로 다음을 대조한다:
   - RED Phase의 테스트 시나리오가 모두 작성·실행되었는지
   - GREEN Phase의 구현 항목이 모두 구현되었는지
3. 누락 항목이 있으면 (최대 2회 반복):
   - 누락 항목에 대해 sd-ref-tdd 가이드에 따라 TDD 사이클을 수행한다
   - 1번으로 돌아가 다시 대조한다
4. 2회 반복 후에도 누락이 남아있으면 남은 항목을 사용자에게 보고한다
5. 모든 항목이 충족되면 5단계로 진행한다

## 5. 완료 안내

모든 작업의 TDD 사이클과 자가검토가 완료되면 다음을 출력한다:
- 전체 작업의 구현 결과 요약 (작업별 상태)
