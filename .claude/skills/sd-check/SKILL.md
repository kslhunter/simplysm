# 검증 스킬

타입체크/린트/테스트를 단계별로 수행하고 수정한다.

## 작업 흐름

1. 범위를 결정한다. (전체 프로젝트 또는 특정 패키지)

2. **타입체크 단계**
   - 타입체크를 수행한다.
   - 오류가 있으면 모두 수정한 후 다음 단계로 진행한다.

3. **린트 단계**
   - 린트를 `--fix` 옵션으로 수행한다. (fix된 문제는 제외)
   - 오류가 있으면 모두 수정한 후 다음 단계로 진행한다.

4. **테스트 단계**
   - 테스트를 수행한다.
   - 오류가 있으면 모두 수정한다.

## 수정 형식

오류 수정 시 아래 형식으로 출력 후 수정 작업을 수행한다:

```
### 수정 #N: {{문제 제목}}
- 위치: {{파일:라인}}
- 문제: {{현재 상태}}
- 수정: {{수정 내용}}
```
