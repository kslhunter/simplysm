# 주문 취소

## Rules

### Rule 1: 배송 전 주문만 취소 가능

- Scenario: 결제 완료 미배송 주문 취소
  - Given: 주문 상태가 "결제완료"
  - When: 사용자가 주문 취소를 요청
  - Then: 주문 상태가 "취소됨"으로 변경됨

- Scenario: 배송 중 주문 취소 시도
  - Given: 주문 상태가 "배송중"
  - When: 사용자가 주문 취소를 요청
  - Then: "배송 중인 주문은 취소할 수 없습니다" 메시지 표시

- Scenario: 이미 취소된 주문 재취소 시도
  - Given: 주문 상태가 "취소됨"
  - When: 사용자가 주문 취소를 요청
  - Then: "이미 취소된 주문입니다" 메시지 표시

### Rule 2: 취소 시 결제 수단별 환불

- Scenario: 카드 결제 주문 취소
  - Given: 결제 수단이 "신용카드"
  - And: 주문 상태가 "결제완료"
  - When: 사용자가 주문 취소를 요청
  - Then: 카드사 환불 API가 호출됨
  - And: "3~5 영업일 내 환불 예정" 메시지 표시

- Scenario: 포인트 결제 주문 취소
  - Given: 결제 수단이 "포인트"
  - And: 주문 상태가 "결제완료"
  - When: 사용자가 주문 취소를 요청
  - Then: 포인트가 즉시 복원됨
  - And: "포인트가 복원되었습니다" 메시지 표시

## Decision Tables

| 주문상태 | 결제수단 | → 취소 가능 | → 환불 방식 |
|---------|---------|------------|------------|
| 결제완료 | 카드 | Y | 카드사 환불 |
| 결제완료 | 포인트 | Y | 즉시 복원 |
| 배송중 | - | N | - |
| 취소됨 | - | N | - |

## Constraints

- 취소 요청은 주문자 본인만 가능
- 환불 금액은 결제 금액과 동일 (부분 취소 미지원)
