# 코드 검토

## 작업 흐름

1. 범위 결정: 프로젝트/패키지 범위를 설정한다.
2. 아키텍처 검토: 지침에 따라 구조를 검토한다.
3. 지적 검증: 검토 결과중 수정이 권장되는 내용만 추려 `sd-review-validate`스킬을 활용하여 검증한다.
4. 수정 계획: 수정 계획을 수립한다.
5. 계획 적용: 수정 계획을 수행한다. (각 수행에 대한 설명을 먼저 사용자에게 제시한후, 바로 수행한다.)
6. 결과 출력: 수행 결과를 출력한다.

## 검토 지침

- **설계 원칙**: SOLID 원칙 위반, 관심사 분리 미흡
- **의존성**: 순환 의존성, 잘못된 의존성 방향
- **결합도/응집도**: 모듈 간 과도한 결합, 단일 책임 위반
- **일관성**: 기존 패턴과 다른 구현, 유사 기능의 상이한 방식
- **확장성**: 변경 시 기술 부채 발생 가능성
