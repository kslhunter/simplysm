# sd-simplify: 코드 품질 검토 및 개선

수정된 파일을 reuse, quality, efficiency 3가지 관점으로 검토하고, 문제 발견 시 직접 수정한다.

**기능 보존 원칙:** 다음은 절대 변경하지 않는다:
- 함수/메서드의 시그니처(매개변수, 반환 타입)
- 외부에서 관찰 가능한 동작(반환값, 부수효과, 호출 순서)
- API 계약(인터페이스, 타입, export)

## 1. 인자 파싱

`$ARGUMENTS`에서 파일 경로 목록을 추출한다.
- 비어있으면 AskUserQuestion으로 파일 경로 목록을 요청한다

## 2. 대상 파일 읽기

파일 목록의 각 파일을 Read한다.

## 3. 3관점 검토

다음 3가지 관점으로 코드를 검토하고, 개선이 필요한 부분을 수정한다.

### Reuse (재사용성)
- 중복 코드를 기존 유틸리티/헬퍼로 대체할 수 있는지
- 반복 패턴을 공통화할 수 있는지

### Quality (품질)
- 네이밍이 의도를 명확히 전달하는지
- 함수/메서드의 책임이 단일한지
- 가독성을 해치는 구조가 있는지

### Efficiency (효율성)
- 불필요한 연산/할당이 있는지
- 더 적합한 데이터 구조나 알고리즘이 있는지

## 4. 완료

- 수정이 없으면 별도 보고 없이 종료한다
- 수정이 있으면 수정된 파일 목록만 간략히 출력한다
