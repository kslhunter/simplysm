# 코드 검토 스킬

CLAUDE.md 의 지침 기반으로 코드를 검토한다.

## 사용법

```
/sd-review [경로]
```

- 경로 미지정 시: 변경된 파일들 검토 (git status 기준)
- 경로 지정 시: 해당 경로만 검토

## 검토 원칙

1. **검색 먼저, 판정 나중**: 각 지침마다 반드시 Grep/Glob으로 검색을 수행하고 결과를 보여준 후 판정한다.
2. **추측 금지**: 검색하지 않은 항목은 "✅ 통과"가 아니라 "⚠️ 미확인"으로 표시한다.
3. **증거 기반**: 위반 사항은 파일:라인 형식으로 정확한 위치를 명시한다.

## 작업 흐름

1. **CLAUDE.md 읽기**
   - 프로젝트 루트의 CLAUDE.md 파일을 읽는다.
   - 각 섹션에서 지침등 코드에서 체크할 부분을 빠짐없이 추출한다.

2. **체크리스트 생성**
   - CLAUDE.md에서 추출한 지침들을 빠짐없이 체크리스트로 만든다.
   - 각 지침에 대해 적절한 검색 패턴을 결정한다.

3. **범위 결정**
   - 인자가 있으면 해당 경로
   - 없으면 사용자에게 인자 요청

4. **지침별 검색 수행**
   - 각 체크리스트 항목마다 Grep/Glob 실행
   - 검색 결과를 그대로 출력

5. **결과 정리**
   ```
   ## 검토 결과

   | 지침 | 상태 | 비고 |
   |------|------|------|
   | {{지침명}} | ✅/❌/⚠️ | 검색 결과 요약 |
   | ... | ... | ... |

   ## 위반 사항

   ### #1. {{문제 제목}}
   - 위치: {{파일:라인}}
   - 문제: {{현재 상태}}
   - 검색 결과: {{실제 grep 결과}}
   - 수정: {{수정 방법}}
   ```

## 상태 표시 기준

- ✅ 통과: 검색 결과 없음 (위반 없음)
- ❌ 위반: 검색 결과 있음 (위반 발견)
- ⚠️ 미확인: 검색하지 않음 또는 해당 없음

## 주의 사항

- 검색 결과가 없어도 "통과"라고 단정짓기 전에 검색 패턴이 올바른지 재확인한다.
- 대상 경로에 따라 해당하지 않는 지침은 "해당 없음"으로 표시한다. (예: solid 패키지가 아닌데 SolidJS 지침)
