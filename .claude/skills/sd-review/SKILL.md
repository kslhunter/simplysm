# 검토 스킬

프로젝트/패키지의 문제점 및 개선점을 종합 검토한다.

## 작업 흐름

### 1. 범위 결정

- 명시 없음 → 전체 프로젝트
- 패키지명 명시 → 해당 패키지

### 2. 관점별 병렬 검토

Task agent를 사용하여 4개 관점에 대해 **병렬로 실행**한다.

#### 2-1. 코드 검토

```
**코딩지침:** `.claude/rules/TYPESCRIPT_RULES.md`
**검토지침:** `references/sd-review-validate-guide.md`

{{대상}}에서 코드 검토:
- 에러 처리, 엣지 케이스 누락, 타입 안전성 등 코드 품질 전반
- 직접 구현 대신 표준 기능 또는 외부 npm 라이브러리로 대체 가능 여부
- simplysm 패키지 기능 우선 사용 권장 (필요시 의존성 추가)
- 코딩 지침(TYPESCRIPT_RULES.md) 준수 여부
- 프로젝트 내 유사 구현이 있으면 재사용 권장
- deprecated API 사용 여부
- 성능 최적화 가능 여부
```

#### 2-2. 아키텍처 검토

```
**코딩지침:** `.claude/rules/TYPESCRIPT_RULES.md`
**검토지침:** `references/sd-review-validate-guide.md`

{{대상}}에서 아키텍처 검토:
- 패키지 간 의존성 구조 적절성
- 순환 의존성 여부
- 계층 구조 및 책임 분리
- 확장성 및 유지보수성
```

#### 2-3. 테스트 검토

```
**코딩지침:** `.claude/rules/TYPESCRIPT_RULES.md`
**검토지침:** `references/sd-review-validate-guide.md`

{{대상}}에서 테스트 검토:
- 기존 테스트가 실제 기능을 충분히 커버하는지 확인
- 누락된 엣지 케이스나 시나리오에 대한 테스트 추가 필요 여부
- 단위테스트와 통합테스트 모두 확인
```

#### 2-4. 문서화 검토

```
**코딩지침:** `.claude/rules/TYPESCRIPT_RULES.md`
**검토지침:** `references/sd-review-validate-guide.md`

{{대상}}에서 문서화 검토:
- 공개 API에 대한 TSDoc 존재 여부
- README.md 내용의 정확성 및 최신성
- 복잡한 로직에 대한 주석 적절성
- 문서화 위치: 코드 관련 → TSDoc (우선) / 구조적·설계 관련 → README.md
- 코드관련 주석중 사용자가 알아야 할것은 반드시 TSDoc에 작성되어야함. 사용자는 몰라도 되는 내용은 일반 주석으로 작성 
```


### 3. 결과 취합 및 재검토

#### 취합

- 2단계 결과를 하나로 병합
- 동일 위치에 대한 중복 지적 제거
- 유사한 제안 통합

#### 지적별 병렬 재검토

비슷한 지적(파일/폴더)끼리 묶고, Task agent를 사용하여 각 그룹을 **병렬로 실행**한다.

```
다음 지적 사항이 유효한지 검증하라:
{{지적내용}}

**코딩지침:** `.claude/rules/TYPESCRIPT_RULES.md`
**검토지침:** `references/sd-review-validate-guide.md`

**출력 형식:**
유효: true/false
제외사유: (false인 경우만) 제외 기준 중 해당 항목
지적내용: (true인 경우) 원본 또는 수정된 지적내용
```

### 4. 결과 출력

재검토 결과를 취합하여 중요도순으로 결과 출력:

```markdown
## {{대상}} 검토 결과

### 제외 사항

| 제목 | 설명 | 제외이유 |
|------|------|----------|

### 문제점/개선점

**1. 문제 제목 (`위치`)**
- **문제**: {{현재 상태 설명}}
- **제안**: {{구체적 수정 방안}}
```

- `{{대상}}`: 프로젝트 혹은 패키지명
- `{{위치}}`: `파일명:라인번호`, `파일명:시작-끝`, `패키지명`, `여러 파일` 등 상황에 맞게 표기
- `{{구체적 수정 방안}}`: A또는 B라는 식의 여러가지 제안을 하는경우, 무엇이 권장되는지 반드시 표기

**주의:** 사용자의 피드백 없이 임의로 수정진행 절대 금지
