# 코드 검토

## 작업 흐름

1. 범위 결정: 프로젝트/패키지 범위를 설정한다.
2. 테스트 검토: 지침에 따라 테스트를 검토한다.
3. 지적 검증: 검토 결과중 수정이 권장되는 내용만 추려 `sd-review-validate`스킬을 활용하여 검증한다.
4. 수정 계획: 수정 계획을 수립한다.
5. 계획 적용: 수정 계획을 수행한다. (각 수행에 대한 설명을 먼저 사용자에게 제시한후, 바로 수행한다.)
6. 결과 출력: 수행 결과를 출력한다.

## 검토 지침

- **테스트 커버리지**: 기존 테스트가 실제 기능을 충분히 커버하는지 확인
- **엣지 케이스**: 누락된 엣지 케이스나 시나리오에 대한 테스트 추가 필요 여부
- **단위 테스트**: 개별 함수/클래스에 대한 테스트 적절성
- **통합 테스트**: 모듈 간 상호작용 테스트 적절성
- **테스트 품질**: 테스트 코드의 가독성, 유지보수성, 중복 여부
