# 코드 검토

## 작업 흐름

1. 범위 결정: 프로젝트/패키지 범위를 설정한다.
2. 문서화 검토: 지침에 따라 문서화를 검토한다.
3. 지적 검증: 검토 결과중 수정이 권장되는 내용만 추려 `sd-review-validate`스킬을 활용하여 검증한다.
4. 수정 계획: 수정 계획을 수립한다.
5. 계획 적용: 수정 계획을 수행한다. (각 수행에 대한 설명을 먼저 사용자에게 제시한후, 바로 수행한다.)
6. 결과 출력: 수행 결과를 출력한다.

## 검토 지침

- **TSDoc**: 공개 API에 대한 TSDoc 존재 여부 및 품질
- **README**: README.md 내용의 정확성 및 최신성
- **주석 적절성**: 복잡한 로직에 대한 주석 적절성
- **문서화 위치**:
    - 코드 관련 내용 중 사용자가 알아야 할 것: TSDoc (우선)
    - 코드 관련 내용 중 내부 로직: 일반 주석
    - 구조적/설계 관련 내용: README.md
- 직역등 어색한 표현을 확인, 수정한다.
- 로직이 복잡한 경우, TSDoc과 주석을 적극 활용한다.
