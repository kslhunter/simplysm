# Git 커밋 메시지 자동 생성

변경사항을 분석하여 SIMPLYSM 프로젝트 컨벤션에 맞는 커밋 메시지를 자동으로 생성하고 커밋합니다.

## 목적

- **정확한 변경사항 파악**: staged 변경사항을 분석하여 무엇이 바뀌었는지 정확히 이해
- **일관된 커밋 스타일**: 프로젝트의 기존 커밋 스타일과 CLAUDE.md 규칙을 준수
- **의미 있는 메시지**: 파일 단위가 아닌 기능/목적 단위로 설명
- **안전한 커밋**: 사용자 확인 후 커밋 실행

## 커밋 메시지 포맷

```
제목: 전체 변경을 아우르는 한 줄 요약 (50자 이내)

- [패키지명] 변경 내용 1 (기능/목적 중심)
- [패키지명] 변경 내용 2
- [공통] 전체 프로젝트 관련 변경사항
```

## 작성 규칙

### 필수 규칙
- **한국어** 작성
- **능동태** 사용 (예: "추가함", "수정함", "제거함", "개선함")
- **패키지별 그룹핑**: 같은 패키지의 변경사항은 함께 묶어서 설명
- **기능 단위 설명**: 개별 파일이 아닌 기능/목적 중심으로 설명
- **명확성**: 변경 이유와 목적이 드러나도록 작성

### 제목 작성 가이드
- 전체 변경의 핵심을 한 문장으로 요약
- 50자 이내 (한글 기준)
- 구체적이면서 간결하게

### 본문 작성 가이드
- 각 항목은 독립적으로 읽어도 이해 가능하도록
- 기술적 세부사항보다는 변경 목적과 의도 중심
- 여러 파일에 걸친 변경은 하나의 기능으로 통합

## 실행 절차

### 1. 변경사항 스테이징
먼저 `git status`로 현재 상태를 확인합니다. unstaged 변경사항이 있다면 항상 `git add .`로 스테이징합니다.

### 2. 컨텍스트 수집
다음 정보를 수집하여 커밋 메시지 작성에 활용합니다:

- **최근 커밋 스타일**: `git log -n 5 --oneline` 또는 `git log -n 3`으로 프로젝트의 커밋 메시지 스타일 파악
- **변경 파일 요약**: `git diff --staged --stat`으로 어떤 파일이 변경되었는지 확인
- **상세 변경사항**: `git diff --staged`로 실제 코드 변경 내용 분석
- **파일 추가/삭제**: 새로 추가되거나 삭제된 파일 확인

### 3. 커밋 메시지 생성
수집한 정보를 바탕으로 위의 포맷과 규칙에 맞는 커밋 메시지를 생성합니다:

- CLAUDE.md의 프로젝트 구조와 패키지 설명 참고
- 최근 커밋들의 스타일과 용어 일관성 유지
- 변경 파일 경로에서 패키지명 추출 (`packages/{패키지명}/`)
- diff 내용을 분석하여 변경 목적 파악

### 4. 사용자 확인
생성된 커밋 메시지를 사용자에게 보여주고 확인을 받습니다. 사용자가 수정을 원하면 피드백을 반영합니다.

### 5. 커밋 실행
사용자 승인 후 `git commit -m "커밋메시지"`를 실행합니다.

## 주의사항

- **변경사항 없음**: staged 변경이 없으면 커밋하지 않고 안내 메시지 출력
- **너무 큰 변경**: 변경사항이 너무 많으면 분리 커밋을 제안
- **민감한 정보**: 커밋 메시지에 API 키, 비밀번호 등 민감 정보 포함 금지
- **WIP 커밋**: 작업 중인 불완전한 코드는 명시적으로 "WIP:" 접두사 사용

## 사용 예시

```bash
# 기본 사용
/sd-commit

# 실행 흐름
# 1. git status 확인
# 2. 필요시 git add
# 3. 변경사항 분석
# 4. 커밋 메시지 생성 및 확인
# 5. 사용자 승인 후 git commit
```
