# /sd:plan - 구현 계획서 작성

$ARGUMENTS

## 트리거

- 새 기능 구현 전 계획 수립
- 리팩토링/마이그레이션 계획
- 버그 수정 전 영향 분석

> 📖 유형별 Plan 예시: [plan-examples.md](../../references/plan-examples.md)

## 인수 파싱

자연어에서 추출:

| 요소 | 예시 |
|------|------|
| 요구사항 | 전체 문장 |
| 대상 패키지 | `core-common에`, `orm-node 패키지` |
| 저장 경로 | `docs/plans에 저장` |

## 출력 파일

| 조건 | 경로 |
|------|------|
| 경로 언급 | `{경로}/PLAN_{계획명}.md` |
| 패키지 언급 | `packages/{name}/PLAN_{계획명}.md` |
| 기본 | `PLAN_{계획명}.md` (루트) |

## 실행 절차

### 1. 컨텍스트 파악 (병렬 실행)

- `CLAUDE.md` 읽기
- 패키지 구조 분석
- 기술 스택 확인

### 2. 요구사항 분석

- 작업 항목으로 분해
- 모호한 부분 → 사용자에게 질문
- **복잡한 요구사항**: `sequential-thinking`으로 단계별 분해

### 3. 코드베이스 탐색

- **대규모 탐색**: `warpgrep_codebase_search`로 자연어 기반 코드 검색 (예: "인증 처리하는 부분 찾아줘")
- 세부 검색: Glob, Grep으로 특정 파일/패턴 검색
- 기존 패턴 파악, 의존성 분석
- 수정 대상 파일 목록화
- **외부 라이브러리 도입 시**: `context7`로 최신 문서 조회하여 API 사용법 확인

### 4. 계획서 작성

**검증 배치 기준** (문제 발생 소지 판단):

| 위치 | 검증 포함 조건 |
|------|----------------|
| Phase 완료 시 | 해당 Phase에 문제 발생 소지가 있을 때 |
| Plan 완료 시 | 전체적으로 문제 발생 소지가 있을 때 |

**문제 발생 소지 예시**:
- 기존 API 시그니처/타입 변경
- 공유 모듈 수정
- 여러 패키지에 걸친 변경
- 런타임 동작 변경

```markdown
# Plan: {제목}

## 요구사항
{원본 요구사항}

## 분석 결과

### 영향 범위
| 패키지 | 파일 | 변경 유형 |
|--------|------|-----------|
| {name} | {path} | 신규/수정/삭제 |

### 주의사항
- 브레이킹 체인지, 테스트 필요 영역

## 구현 단계

### Phase 1: {단계명}
**목표**: ...

- [ ] 작업 1 (`path/to/file.ts`)
- [ ] 작업 2

**검증**: `/sd:check {패키지}` 또는 수동 확인 방법 (문제발생 소지가 있는 Phase만)

## 최종 검증
- [ ] 전체 타입 체크 (phase내에서 안한 경우)
- [ ] 전체 테스트 (phase내에서 안한 경우)
- [ ] 통합 테스트/E2E (문제발생 소지가 있는 경우)

## 문서 업데이트
- [ ] CLAUDE.md / README.md 업데이트 (`/sd:docs`)

## 코드 리뷰
- [ ] 최종 코드 리뷰 (`/sd:review`) - 권장
```

### 5. 파일 저장 및 안내

- 계획서를 지정 경로에 저장
- 저장된 파일 경로 및 요약 안내
- **구현 자동 시작 금지**

### 6. TodoWrite 등록

- Phase별 주요 작업 항목을 TodoWrite에 등록 (pending 상태)
- `/sd:task`에서 이어서 작업할 수 있도록 상태 추적

## 예외 처리

| 상황 | 대응 |
|------|------|
| 기존 PLAN 파일 존재 | 덮어쓸지 / 다른 이름으로 저장할지 확인 |
| 요구사항 모호 | 사용자에게 질문 후 진행 |
| 패키지 없음 | 새 패키지 생성 여부 확인 |
| 의존성 순환 감지 | 경고 후 대안 제시 |

## 주의사항

- **구현 금지**: 계획서 작성만. 구현하지 않음
- **코드 기반**: 추측 금지. 실제 코드 분석 결과만 기재

## 사용 예시

```bash
/sd:plan ORM에 트랜잭션 지원 추가
/sd:plan service-server에 사용자 인증 기능 구현해줘
/sd:plan 레거시 API 마이그레이션 계획 docs/plans에 저장
```
