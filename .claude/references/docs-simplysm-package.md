# simplysm 패키지 문서 작성 가이드

`/sd:docs {패키지명}` 실행 시 (simplysm 저장소) 참조하는 가이드입니다.

---

## 1. 작성 목적

| 문서 | 목적 | 대상 독자 |
|------|------|-----------|
| **CLAUDE.md** | 패키지 개발/수정 시 필요한 가이드 | Claude Code |
| **README.md** | 패키지 사용법 안내 | 개발자 (사용자) |

---

## 2. 분석 절차

### 소스코드 분석

1. `package.json` - 이름, 설명, 의존성
2. `src/index.ts` - public API 확인
3. 기존 문서 - 있으면 참고

### 분석 방법

- **소규모** (파일 20개 미만): 직접 Read
- **대규모**: `warpgrep_codebase_search`로 핵심 로직 탐색

### 외부 의존성

- 외부 라이브러리 사용 시 `context7`로 문서 조회

---

## 3. CLAUDE.md 작성

### 동작

1. 기존 CLAUDE.md 확인
2. 없으면 신규 작성, 있으면 검토/개선
3. 소스코드 분석 결과 반영

### 섹션 구조

| 섹션 | 필수 | 내용 |
|------|:----:|------|
| 헤더 | ✅ | 패키지명, 루트 CLAUDE.md 링크 안내 |
| 한 줄 설명 | ✅ | 패키지 역할 요약 |
| 목적 | ✅ | 역할, 책임, 다른 패키지와의 관계 |
| 구조 | ✅ | 디렉토리 구조 및 역할 |
| 핵심 개념 | ✅ | 주요 개념 설명 |
| 주요 API | ✅ | 핵심 함수/클래스 표 |
| 개발 규칙 | 선택 | 패키지 특화 규칙, 제약사항 |
| 의존성 | ✅ | 내부(@simplysm/*), 외부 패키지 |
| 주의사항 | 선택 | 흔한 실수, 피해야 할 패턴 |

### 헤더 템플릿

```markdown
# {패키지명}

**이 문서는 Claude Code가 {패키지명} 패키지를 개발/수정할 때 참고하는 가이드입니다.**
**프로젝트 루트의 [CLAUDE.md](../../CLAUDE.md)를 함께 확인하세요.**

---
```

### 작성 가이드

- **정확성**: 실제 코드와 일치하는 내용
- **핵심 위주**: 모든 API가 아닌 주요 API 중심
- **의존성 명시**: 내부/외부 의존성 명확히 구분
- **실용성**: 개발 시 자주 참고할 내용 위주

### 체크리스트

- [ ] 헤더에 루트 CLAUDE.md 링크 포함
- [ ] 구조가 실제 디렉토리와 일치
- [ ] 주요 API가 index.ts export와 일치
- [ ] 의존성이 package.json과 일치
- [ ] 핵심 개념이 이해하기 쉽게 설명됨

---

## 4. README.md 작성

### 동작

1. 기존 README.md 확인
2. 없으면 신규 작성, 있으면 검토/개선
3. 사용자 관점에서 작성

### 섹션 구조

| 섹션 | 필수 | 내용 |
|------|:----:|------|
| 헤더 | ✅ | @simplysm/{패키지명}, CLAUDE.md 링크 안내 |
| 설명 | ✅ | 패키지 소개 |
| 설치 | ✅ | yarn add 명령어 |
| 요구사항 | 선택 | Node.js 버전 등 |
| 사용법 | ✅ | 기본 사용 예시 코드 |
| API | ✅ | 주요 함수/클래스 설명 |
| 관련 패키지 | 선택 | 함께 사용하는 패키지 |
| 라이선스 | ✅ | MIT |

### 헤더 템플릿

```markdown
# @simplysm/{패키지명}

**이 문서는 패키지 사용자를 위한 가이드입니다.**
**개발자용 가이드는 [CLAUDE.md](./CLAUDE.md)를 참고하세요.**

---
```

### 작성 가이드

- **사용자 관점**: 설치 → 사용 순서로 작성
- **예시 코드**: 실제 동작하는 TypeScript 코드
- **API 문서**: 파라미터, 반환값 명시
- **간결함**: 상세 내용은 코드 주석이나 TSDoc으로

### 체크리스트

- [ ] 설치 명령어가 정확함 (`yarn add @simplysm/{패키지명}`)
- [ ] 예시 코드가 실제 동작함
- [ ] API 설명이 실제 시그니처와 일치
- [ ] 관련 패키지 링크가 유효함
