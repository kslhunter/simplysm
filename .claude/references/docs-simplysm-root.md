# simplysm 루트 문서 작성 가이드

`/sd:docs --root` 실행 시 (simplysm 저장소) 참조하는 가이드입니다.

---

## 1. 작성 목적

| 문서 | 목적 | 대상 독자 |
|------|------|-----------|
| **CLAUDE.md** | Claude Code 개발 시 필요한 규칙, 인덱싱 정보 | Claude Code |
| **README.md** | 프레임워크 소개 및 사용 안내 | 개발자 (사용자) |

---

## 2. CLAUDE.md 작성

### 동작

1. 기존 CLAUDE.md 읽기
2. 분석:
   - 필수 섹션 누락 여부
   - 오래된 정보 (버전, 패키지 목록 등)
   - 개선 가능한 부분
3. 변경 제안 또는 업데이트

### 섹션 구조

| 섹션 | 필수 | 내용 |
|------|:----:|------|
| 필수 규칙 (CRITICAL) | ✅ | 언어, bash, 파일 경로 규칙 |
| 프로젝트 개요 | ✅ | 기술 스택, 브라우저 환경 |
| 아키텍처 | ✅ | 패키지 의존성 다이어그램 |
| 코드 스타일 | ✅ | 네이밍, TypeScript, Angular 규칙 |
| 모노레포 규칙 | ✅ | 파일 구조, Import 규칙 |
| Commands | ✅ | /sd:* 커맨드 목록 및 링크 |
| 설정 파일 | ✅ | 주요 설정 파일 목록 |
| 패키지별 가이드 | ✅ | 각 패키지 CLAUDE.md 링크 |
| 마이그레이션 | 선택 | 활성/레거시 패키지 상태 |

### 작성 가이드

- **일관성**: 기존 문서의 스타일과 포맷 유지
- **최신화**: 버전, 패키지 목록은 실제 상태 반영
- **간결함**: 상세 내용은 패키지별 CLAUDE.md로 분리
- **링크**: 관련 파일/문서는 상대 경로로 연결

### 체크리스트

- [ ] 모든 활성 패키지가 "패키지별 가이드"에 링크됨
- [ ] 기술 스택 버전이 실제와 일치
- [ ] Commands 섹션이 실제 커맨드와 일치
- [ ] 마이그레이션 상태가 최신임
- [ ] 레거시 패키지 수정 금지 언급됨

---

## 3. README.md 작성

### 동작

1. 기존 README.md 확인
2. 없으면 신규 작성, 있으면 검토/개선
3. 변경 제안 또는 업데이트

### 섹션 구조

| 섹션 | 필수 | 내용 |
|------|:----:|------|
| 프로젝트 소개 | ✅ | 한 줄 설명, 특징 |
| 설치 | ✅ | 요구사항, 설치 방법 |
| 시작하기 | ✅ | 빠른 시작 가이드 |
| 패키지 목록 | ✅ | 각 패키지 설명 표 |
| 기여 | 선택 | 기여 가이드 |
| 라이선스 | ✅ | MIT |

### 작성 가이드

- **사용자 관점**: 개발자가 빠르게 시작할 수 있도록
- **예시 코드**: 실제 동작하는 코드 스니펫 포함
- **패키지 링크**: 각 패키지 README.md로 연결

### 체크리스트

- [ ] 설치 명령어가 정확함
- [ ] 패키지 목록이 활성 패키지와 일치
- [ ] 예시 코드가 동작함
- [ ] 버전 정보가 최신임
