# Eval: sd-option-scoring

## 행동 Eval

### 시나리오 1: 수정 방안 제시
- 입력: "이 프로젝트에서 상태 관리를 Redux로 할지 Zustand로 할지 고민이야"
- 체크리스트:
  - [ ] 각 선택지에 10점 만점 점수가 포함되어 있다
  - [ ] 각 선택지에 장단점 또는 트레이드오프가 기술되어 있다
  - [ ] "수행 안 함" 또는 "현행 유지" 또는 "도입하지 않음" 옵션이 포함되어 있다
  - [ ] "수행 안 함" 옵션에도 점수가 매겨져 있다
  - [ ] `---` 구분선 후 사용자에게 선택을 요청하는 텍스트가 출력된다

### 시나리오 2: 단일 방안만 있는 경우
- 입력: "console.log를 Logger로 교체하고 싶어"
- 체크리스트:
  - [ ] 교체 방안에 10점 만점 점수가 포함되어 있다
  - [ ] "교체하지 않음" 옵션이 포함되어 있다
  - [ ] "교체하지 않음" 옵션에도 장단점과 점수가 매겨져 있다

## 안티패턴 Eval

- [ ] 선택지를 점수 없이 제시하지 않는다
- [ ] "수행 안 함" 옵션 없이 선택지를 제시하지 않는다
- [ ] 30점 만점 등 10점 만점이 아닌 점수 체계를 사용하지 않는다
