# Eval: sd-option-scoring

## 행동 Eval

### 시나리오 1: 수정 방안 제시
- 입력: "이 프로젝트에서 상태 관리를 Redux로 할지 Zustand로 할지 고민이야"
- 체크리스트:
  - [ ] 각 선택지에 10점 만점 점수가 포함되어 있다
  - [ ] 각 선택지에 장단점 또는 트레이드오프가 기술되어 있다
  - [ ] "수행 안 함" 또는 "현행 유지" 또는 "도입하지 않음" 옵션이 포함되어 있다
  - [ ] "수행 안 함" 옵션에도 점수가 매겨져 있다
  - [ ] `---` 구분선 후 사용자에게 선택을 요청하는 텍스트가 출력된다

### 시나리오 2: 단일 방안만 있는 경우
- 입력: "console.log를 Logger로 교체하고 싶어"
- 체크리스트:
  - [ ] 교체 방안에 10점 만점 점수가 포함되어 있다
  - [ ] "교체하지 않음" 옵션이 포함되어 있다
  - [ ] "교체하지 않음" 옵션에도 장단점과 점수가 매겨져 있다

### 시나리오 3: 복수 결정사항 처리

- 입력: "이 프로젝트에 CI/CD를 도입하고 싶어. 플랫폼은 GitHub Actions vs GitLab CI 중 고민이고, 배포 전략은 블루그린 vs 카나리 중 고민이야"
- 체크리스트:
  - [ ] 두 결정사항(CI/CD 플랫폼, 배포 전략)을 한 번에 모두 나열하지 않는다
  - [ ] 첫 번째 결정사항의 선택지를 먼저 제시한다
  - [ ] 첫 번째 결정사항에 대한 사용자 선택 후 두 번째 결정사항으로 넘어간다
  - [ ] 남은 결정사항 건수를 안내한다 (예: "1/2건 완료")

## 안티패턴 Eval

- [ ] 점수 없이 선택지를 제시한다
- [ ] "수행 안 함" 옵션 없이 선택지를 제시한다
- [ ] 30점 만점 등 10점 만점이 아닌 점수 체계를 사용한다
- [ ] 복수 결정사항을 한꺼번에 나열한다
