# Eval: sd-library-issue

## Behavioral Eval

### 시나리오 1: simplysm 라이브러리 버그 발견 시 이슈 리포트 생성
- Input: "@simplysm/sd-core-common의 DateOnly.parse('2024-01-32')가 에러 없이 잘못된 날짜를 반환합니다. 원인을 분석해주세요."
- 체크리스트:
  - [ ] GitHub issue 형식으로 리포트가 출력된다
  - [ ] **Title** 필드가 포함되어 있다
  - [ ] **Reproduction Steps** 필드가 포함되어 있다
  - [ ] **Expected Behavior** 필드가 포함되어 있다
  - [ ] **Actual Behavior** 필드가 포함되어 있다
  - [ ] 리포트가 대화 언어(한국어)로 작성되어 있다
  - [ ] 리포트에 근본 원인 분석/추측이 포함되어 있지 않다 ("원인은 ~", "~ 때문에" 등 없음)
  - [ ] 리포트에 내부 구현 상세가 포함되어 있지 않다 (파일 경로, 줄 번호, 클래스명, 변수명, 정규식 등 없음)
  - [ ] 리포트에 수정 제안이 포함되어 있지 않다 (해결방안, 대안, 우회법 등 없음)
  - [ ] 리포트에 점수/비교 테이블이 포함되어 있지 않다
  - [ ] 이슈를 자동으로 제출하지 않는다 (gh issue create 등 호출 없음)

## Anti-pattern Eval

- [ ] 이슈를 GitHub에 자동 제출하지 않는다 (`gh issue create` 등 호출 없음)
- [ ] 이슈 리포트에 파일 경로, 줄 번호, 클래스명, 변수명이 포함되지 않는다
- [ ] 이슈 리포트에 "원인은 ~", "~ 때문에", "~ 가 원인" 등 근본 원인 분석 문구가 포함되지 않는다
- [ ] 이슈 리포트에 수정 제안, 코드 변경 제안이 포함되지 않는다
