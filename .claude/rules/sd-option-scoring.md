# 선택지 제시 규칙

사용자에게 선택지를 제시하는 **모든 상황**에서 아래 규칙을 따른다.

## 복수 결정사항 처리

- **CRITICAL: 결정할 사항이 여러 개일 경우, 절대 한 번에 모든 질문을 나열하지 않는다. 반드시 한 번에 하나의 결정사항만 제시하고, 사용자 선택 후 다음으로 넘어간다.**
- 첫 번째 결정사항의 선택지를 제시 → Ask → 사용자 선택 후, 다음 결정사항으로 넘어간다
- 이전 선택 결과에 따라 후속 결정사항이 변경·추가·삭제될 수 있으므로, 매 단계마다 남은 결정사항을 재평가한다
- 남은 결정사항이 몇 건인지 알려준다 (예: "1/3건 완료")
- **안티패턴:** 여러 질문을 한꺼번에 나열하는 것은 금지 (예: Q1~Q11을 한 번에 출력)

## 필수 항목

- 각 선택지에 대해 **장단점/트레이드오프/10점 만점 점수**를 포함한다
- **"수행 안 함"** 옵션을 반드시 포함한다 — 수정/변경을 하지 않는 선택지도 동일하게 장단점과 점수를 매긴다
- 선택지 정보를 텍스트로 출력한 뒤 `---` 구분선을 출력하고, `AskUserQuestion`으로 질문한다

## 점수 기준

- 모든 점수는 **10점 만점**이다
- LLM이 맥락에 맞는 관점(축)을 자율적으로 선정하여 관점별 점수를 매기고, **평균을 10점 만점으로 표시**한다
  - 구현 비용, 복잡도, 단순성 같은건 관점에 포함하지 않는다. 안정성은 반드시 관점에 포함한다.
